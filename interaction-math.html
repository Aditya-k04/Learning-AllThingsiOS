Math for Interaction
Model the visual as a function; read off rotation, spacing, and bend.

    Math for Interaction Design — Roadmap

    The skill isn't "advanced math." It's a way of seeing: model the visual as a function, then use calculus / linear algebra to extract the geometric thing you need.

    This curriculum exists because of one caption on a wavy carousel:

    "Calculating the derivative of that wave lets you rotate the cells to sit perfectly on the curve. As you pinch, we stretch the amplitude of a sine wave that the cells follow."

    That sentence secretly contains trigonometry (the wave), a derivative (tangent → rotation), and nearby curvature (2nd derivative) and arc length (an integral). Become fluent in those moves and you can place and orient anything on anything. See the full decode in 08-worked-example-wavy-carousel.

    The one mental move

    position   y(x)      →  WHERE a thing is
tangent    y'(x)     →  which way it FACES   (rotation)
curvature  y''(x)    →  how hard it BENDS    (squash / scale)
arc length ∫|y'|dx   →  distance ALONG the path (even spacing)

    Every interaction in this folder is an application of this. Internalize it and specs write themselves.

    01 — Modeling Visuals as Functions

    The single habit that separates "I tweak values until it looks right" from "I can derive exactly what I need."

    The shift

    Stop asking "what animation API do I call?" Ask:

    What function describes this? And what does its derivative / normal / integral give me?

    Once a visual is a function, the whole toolkit of calculus and linear algebra becomes a set of design controls.

    The meta-move (memorize this)

    0th derivative   position    →  WHERE it is
1st derivative   tangent      →  which way it FACES  (rotation, velocity)
2nd derivative   curvature    →  how hard it BENDS   (acceleration, squash)
integral         accumulation →  arc length, area, "summed-up" quantities

    Each one hands you a new geometric handle on the same object. That progression is the "level of thinking" you're chasing.

    Worked micro-example

    Cells follow a wave y(x) = A·sin(ω·x + φ).

    • Where is cell i? → y(x_i)
    • Which way does it face so it sits flush on the curve? → tangent angle θ = atan2(y'(x_i), 1), and y'(x) = A·ω·cos(ω·x + φ)
    • Pinch stretches A → because y' is proportional to A, the tilt updates for free. One parameter, coherent result.

    Full derivation: 08-worked-example-wavy-carousel.

    Why this is the high-leverage skill

    • It makes effects composable: change one parameter, everything downstream follows correctly.
    • It makes them art-directable: you expose the parameters of the function (A, ω, φ) as sliders.
    • It removes guesswork: no more "0.45 felt right" — you derive the relationship.

    Try it

    In Desmos: plot y = A sin(b x + c) with sliders for A, b, c. Then plot its derivative A b cos(b x + c) underneath. Drag A and watch both change together. That coupling is the whole idea.

    02 — Trigonometry & the Unit Circle

    Not "SOH-CAH-TOA." The one idea that unlocks waves, rotation, and oscillation at once.

    The core insight

    cos and sin are the x and y of a point travelling around the unit circle.

    point on circle at angle t  =  (cos t, sin t)

    So a wave is circular motion projected onto a line. Oscillation, rotation, and waves are the same phenomenon seen from different angles. Once you see this, half of "creative math" collapses into one picture.

    Anatomy of a wave

    y(x) = A · sin(ω·x + φ) + k
        │        │     │    └ vertical offset (baseline)
        │        │     └ phase   (horizontal shift / starting angle)
        │        └ angular frequency (how many radians per unit x → how tight)
        └ amplitude (height)
    • Amplitude A — how tall. (Pinch-to-stretch changes this.)
    • Frequency ω — how many wiggles. Period = 2π / ω.
    • Phase φ — slides the wave sideways; also = "where on the circle it starts."

    Things you can build immediately

    • Pulsing / breathing: scale = 1 + A·sin(ω·t) driven by time t.
    • Staggered wave across items: give item i a phase φ = i · Δ, so y_i = A·sin(ω·t + i·Δ) → a travelling wave through a list.
    • Circular layout: place item i at angle θ_i = i · 2π/n, position (R·cos θ_i, R·sin θ_i).
    // Travelling wave through a row of dots
for (i, dot) in dots.enumerated() {
    let phase = Double(i) * 0.4
    dot.offsetY = amplitude * sin(omega * time + phase)
}

    Bridge to rotation (important)

    A 2D rotation by angle θ is built entirely from sin/cos:

    [ cos θ  −sin θ ] [x]
[ sin θ   cos θ ] [y]

    So trig ↔ rotation ↔ linear algebra are one topic. Continue in 04-vectors-and-linear-algebra.

    Try it

    Desmos: plot (cos t, sin t) parametrically for t in [0, 2π] → you get a circle. Now plot (t, sin t) → you get the wave. Same sin, two viewpoints.

    03 — Calculus: the Derivative as Tangent

    You don't need symbol-pushing. You need the geometric meaning of three operations.

    Derivative = slope = tangent direction

    The derivative y'(x) is the slope of the curve at x. Slope is a direction. That's why it gives you rotation.

    tangent vector at x  =  (1, y'(x))           // run 1, rise y'
tangent angle        =  atan2(y'(x), 1)      // rotate a cell by this to lie on the curve

    The six derivatives worth memorizing

    d/dx [ sin x ]   = cos x
d/dx [ cos x ]   = −sin x
d/dx [ x^n ]     = n·x^(n−1)
d/dx [ e^x ]     = e^x
d/dx [ c ]       = 0          (constant)
chain rule:  d/dx f(g(x)) = f'(g(x)) · g'(x)

    Chain rule is why d/dx [A·sin(ω x + φ)] = A·ω·cos(ω x + φ) — the inner ω pops out.

    Second derivative = curvature (how hard it bends)

    κ(x) = y''(x) / (1 + y'(x)²)^(3/2)

    Use it to squash or scale an element more on tight bends, or to drive "acceleration feel."

    Integral = accumulation → arc length

    To space items evenly along a curve (not evenly in x, which bunches them on steep parts):

    arc length from a to b  =  ∫_a^b sqrt(1 + y'(x)²) dx

    In code you usually do this numerically (walk the curve in tiny steps, sum segment lengths, then resample at equal distances).

    // Sample a curve, build a cumulative-length table, resample at even spacing
func arcLengthSamples(_ f: (Double)->Double, x0: Double, x1: Double, n: Int) -> [(x: Double, s: Double)] {
    var out: [(Double, Double)] = []
    var s = 0.0
    var prev = CGPoint(x: x0, y: f(x0))
    let dx = (x1 - x0) / Double(n)
    for i in 0...n {
        let x = x0 + Double(i) * dx
        let p = CGPoint(x: x, y: f(x))
        s += hypot(p.x - prev.x, p.y - prev.y)
        out.append((x, s)); prev = p
    }
    return out   // invert s→x to place items at equal arc-length
}

    The progression to keep in your head

    y      WHERE      position
y'     FACING     rotation / velocity   ← the carousel trick
y''    BENDING    curvature / accel
∫      ALONG      arc length / area

    Try it

    Desmos: plot a function and tangent(x0) at a draggable point. Drag it and watch the tangent line ride the curve — that line's angle is your rotation.

    04 — Vectors & Linear Algebra

    The native language of graphics. Two products do most of the work.

    A vector is a direction-with-length

    Position, velocity, normal, offset — all vectors. Learn to think in vectors instead of separate x/y and the code shrinks.

    import simd
let v = SIMD2<Float>(3, 4)
let len = length(v)          // 5
let dir = normalize(v)       // unit vector, length 1

    Dot product = "how aligned are these two directions?"

    a · b = ax·bx + ay·by = |a||b|·cos(angle between them)
    • > 0 → same general direction · 0 → perpendicular · < 0 → opposing.
    • Uses: lighting (N·L), "is the user dragging toward this?", projecting one vector onto another, finding angles.
    let alignment = dot(normalize(drag), normalize(target))  // 1 = straight at it

    Cross product (2D) = "how perpendicular / which way does it turn?"

    In 2D it's a single signed scalar:

    cross(a, b) = ax·by − ay·bx
    • Sign = turn direction (left/right, clockwise/ccw) → winding, "which side of the line."
    • Magnitude = area of the parallelogram → signed area, overlap tests.
    • Perpendicular (normal) of (x, y) is (−y, x) — rotating 90°. This is how you offset along a curve's normal.
    func normal(of tangent: SIMD2<Float>) -> SIMD2<Float> {
    normalize(SIMD2(-tangent.y, tangent.x))   // 90° turn
}

    Rotation is a matrix (and it's just sin/cos)

    R(θ) = [ cos θ  −sin θ ]      rotate point p:  p' = R(θ) · p
       [ sin θ   cos θ ]

    func rotate(_ p: SIMD2<Float>, by t: Float) -> SIMD2<Float> {
    SIMD2(cos(t)*p.x - sin(t)*p.y,
          sin(t)*p.x + cos(t)*p.y)
}

    Translation + rotation + scale compose by multiplying matrices — this is exactly what CGAffineTransform and CATransform3D are. See 15-transform-3d.

    Why this matters for the carousel

    The cell's rotation comes from a tangent vector; its perpendicular offset uses the normal (−y, x); placing it on a circle uses a rotation matrix. Same three tools, everywhere.

    Try it

    Take two draggable vectors in Desmos (or paper). Compute dot and cross by hand as you move them; predict the sign before you calculate. Build the reflex: dot = alignment, cross = turn.

    05 — Parametric Curves & Splines

    Generalize "the wave" to any path — circles, arcs, hand-drawn strokes, Béziers.

    Parametric form: describe motion, not just shape

    Instead of y = f(x), use a parameter t:

    P(t) = ( x(t), y(t) )          t usually in [0, 1]
P'(t) = ( x'(t), y'(t) )       velocity → TANGENT direction

    This handles loops and verticals that y = f(x) can't, and the tangent falls right out of the derivative — same trick as 03-derivative-as-tangent, now on any path.

    tangent angle at t  =  atan2(y'(t), x'(t))
normal              =  (−y'(t), x'(t))   normalized   // see 04

    Examples

    Circle:  P(t) = (R cos 2πt, R sin 2πt)
Wave:    P(t) = (t·W, A sin(ω t))
Spiral:  P(t) = (a·t·cos t, a·t·sin t)

    Bézier curves (the workhorse)

    A cubic Bézier is a blend of 4 control points P0..P3:

    B(t) = (1−t)³P0 + 3(1−t)²t P1 + 3(1−t)t² P2 + t³ P3
    • P0, P3 = endpoints; P1, P2 = "handles" that pull the curve.
    • Its derivative B'(t) (also a Bézier, one degree lower) gives the tangent → orientation along the path.
    • This is what UIBezierPath, CAShapeLayer paths, and the cubic timing curve all are. See 16-cashapelayer.

    Catmull-Rom (curve through points)

    Béziers don't pass through their middle controls; Catmull-Rom splines pass through every point — ideal for "lay cells along these positions smoothly." It's a few lines and converts to Béziers for drawing.

    Placing things along a curve

    1. Sample the curve → build a length table (03-derivative-as-tangent arc length).
    2. Resample at equal arc-length for even spacing.
    3. At each sample, take the tangent → rotate the element; take the normal → offset it.

    This is the wavy-carousel recipe generalized to any path: 08-worked-example-wavy-carousel.

    Try it

    UIBezierPath / Swift Charts / Desmos: drop 4 control points, drag the handles, and render the tangent at t = 0.5. Watch how the handles steer the orientation.

    06 — Interpolation, Easing & Springs

    The small toolkit you reach for in every falloff — UI or shader. (Deep companion: interpolation-and-easing.)

    The five functions to know cold

    lerp(a, b, t)      = a + (b − a)·t                 // blend a→b
inverseLerp(a,b,v) = (v − a) / (b − a)             // "where is v between a and b?" → t
clamp(v, lo, hi)   = max(lo, min(hi, v))
remap(v, a,b, c,d) = lerp(c, d, clamp(inverseLerp(a,b,v), 0, 1))   // map one range to another
smoothstep(e0,e1,x): t = clamp((x−e0)/(e1−e0),0,1); return t·t·(3 − 2t)

    remap + smoothstep are 80% of "make this value drive that effect smoothly." (The Stories-strip edge falloff was exactly smoothstep over a remapped distance.)

    func smoothstep(_ e0: Float, _ e1: Float, _ x: Float) -> Float {
    let t = max(0, min(1, (x - e0) / (e1 - e0)))
    return t * t * (3 - 2 * t)
}

    Easing = reshaping t before you lerp

    An easing curve is just a function [0,1] → [0,1] applied to progress. easeInOut ≈ smoothstep. Cubic timing curves are Béziers (05-parametric-curves-and-splines). See 14-animation-timing.

    Springs = physics instead of a fixed curve

    A spring is a second-order differential equation — which is why it connects back to calculus (03-derivative-as-tangent):

    a = −k·x − c·v          x = displacement from target
                        v = velocity, a = acceleration
                        k = stiffness, c = damping
v += a·dt ;  x += v·dt   // integrate each frame
    • Critical damping (c = 2·sqrt(k)) → fastest settle with no overshoot. (Used in the Perplexity dots' return.)
    • Under-damped → bouncy; over-damped → sluggish.
    • Springs are retargetable mid-flight (no fixed duration) → great for gestures. Full treatment: damped-harmonic-oscillator and the Wave library.
    // One critically-damped step
let damping = 2 * sqrt(stiffness)
let a = -stiffness * x - damping * v
v += a * dt
x += v * dt

    When to use which

    Need Use
    Map a position/scroll to an effect remap + smoothstep
    A→B over a fixed time, designed feel easing curve / Bézier timing
    Gesture-driven, interruptible, physical spring

    Try it

    Build a spring in a playground; expose k and c sliders. Find critical damping by feel, then verify c = 2√k.

    07 — Oscillations, Noise & Organic Motion

    How to make motion feel alive instead of mechanical.

    Damped oscillation = bounce that dies out

    y(t) = A · e^(−λt) · sin(ω t + φ)
        │    │          └ the wiggle
        │    └ exponential decay envelope (λ = decay rate)
        └ initial amplitude

    The e^(−λt) envelope is the secret to natural settling — it's the analytic cousin of the spring in 06-interpolation-easing-and-springs. See damped-harmonic-oscillator.

    Summed sines (Fourier intuition)

    Any periodic motion is a sum of sine waves.

    Layer a few sines at different frequencies/amplitudes and you get rich, non-trivial motion from trivial parts:

    y(t) = Σ  A_i · sin(ω_i · t + φ_i)

    Use incommensurate frequencies (not simple multiples) so it never visibly repeats — that's the "gently alive, never looping" feel (the Perplexity dots' idle drift used exactly this).

    let idle = a1*sin(0.6*t + p1) + a2*sin(0.43*t + p2)   // 0.6, 0.43 don't sync

    Noise = controlled randomness with smoothness

    Plain random() is jittery. Perlin / simplex noise is smooth random — neighboring inputs give nearby outputs. That smoothness is what reads as organic (smoke, flowing fields, hand-tremor).

    • Sample noise over time for wandering motion: offset = noise(seed, t·speed).
    • Sample over space for fields/turbulence: n = noise(x·scale, y·scale).
    • Domain warping (feed noise into the input of more noise) gives liquid, billowing motion. See 15-noise-and-patterns.

    Recipe: "gently alive" element

    position = rest
         + summedSines(t)          // slow breathing      (this note)
         + noise(seed, t)·amp      // organic wander       (this note)
         + springToTarget(...)     // responds to input    (06)

    That layered structure is how the best ambient interactions are built.

    Try it

    In a shader or playground, animate a dot with one sine, then add a second at an unrelated frequency, then add a touch of noise. Feel each layer's contribution.

    08 — Worked Example: The Wavy Carousel

    The full decode of: "the derivative of that wave lets you rotate the cells to sit on the curve; as you pinch, we stretch the amplitude." Everything in this folder, applied once.

    Step 1 — Model the curve as a function

    Cells ride a sine wave along the horizontal axis x:

    y(x) = A · sin(ω·x + φ)

    A = amplitude (the pinch target), ω = frequency, φ = phase. → 02-trigonometry-and-the-unit-circle

    Step 2 — Place each cell

    Cell i sits at some x_i (its scroll position). Its vertical offset:

    y_i = A · sin(ω·x_i + φ)

    Step 3 — Derivative → tangent → rotation

    The slope at x_i (chain rule, 03-derivative-as-tangent):

    y'(x) = A · ω · cos(ω·x + φ)

θ_i = atan2( y'(x_i), 1 )      // tangent angle
cell.rotation = θ_i            // now it lies flush on the curve

    Step 4 — Pinch stretches amplitude (and rotation follows for free)

    Map the pinch gesture's scale onto A:

    A = remap(pinchScale, 1.0, 3.0,  Amin, Amax)    // see 06

    Because y' is proportional to A, increasing amplitude automatically increases the tilt. One parameter drives shape and orientation coherently — that coupling is the craft.

    Step 5 (level-up) — even spacing via arc length

    Equal x spacing bunches cells on the steep parts. Space them by arc length instead (03-derivative-as-tangent):

    s(x) = ∫ sqrt(1 + y'(x)²) dx     // distance along the wave

    Sample the curve, build a length table, resample at equal s. Now cells are visually even no matter how tall the wave gets.

    Step 6 (extra polish) — curvature

    Scale/foreshorten cells slightly by curvature κ = y'' / (1 + y'²)^{3/2} so they "hug" tight bends. → 03-derivative-as-tangent

    The whole thing in pseudo-code

    on gesture or frame:
    A = remap(pinchScale, 1, 3, Amin, Amax)
    for each cell i at x_i:
        y      = A * sin(ω*x_i + φ)
        slope  = A * ω * cos(ω*x_i + φ)
        θ      = atan2(slope, 1)
        cell.position = (x_i, baseline + y)
        cell.rotation = θ
        # optional: cell.scale = 1 - clamp(curvature)*k

    Build it yourself (Week 4–5)

    Do it in SwiftUI first (rotationEffect + offset per cell, driven by a MagnifyGesture's scale), then understand it as a pure function of (x_i, A). This is the function-driven mindset from 09-what-drives-the-value.

    09 — What Drives the Value (the iOS Bridge)

    The math tells you what the value should be. This note tells you how iOS should compute it over time. Same three categories, every interaction.

    The three drivers

    Driver The value is… iOS tools Example
    Input-driven (a pure function) f(scrollOffset) or f(gesture) — no animation visualEffect, scrollTransition, UIScrollView.contentOffset, gesture translation Stories edge-scale; the wavy carousel
    Interpolated (a tween) a start→end the system blends SwiftUI .animation, UIViewPropertyAnimator, CA implicit/explicit a button press, a sheet
    Per-frame (imperative) recomputed each frame CADisplayLink, Metal compute, manual loop Perplexity dots; physics fields

    The carousel and the Stories strip are category 1 — they are functions of gesture/scroll, not animations. That's why "no per-frame effect, apply to the main composition" was the right instinct.

    How to choose

    1. Is the value a direct function of a gesture/scroll position? → pure function (cleanest; prefer this).
    2. Is it "go from A to B with a nice feel"? → tween / spring.
    3. Does it need continuous simulation (particles, fields, frame-precise timing)? → per-frame.

    Why understanding Core Animation matters here

    SwiftUI and UIKit are skins over Core Animation. The math you compute becomes layer transforms that the render server interpolates. Knowing the substrate explains why category-2 tweens behave as they do (model vs presentation layer) and when to drop to category 3.

    • Substrate: Core Animation Roadmap → 01-how-rendering-works, 15-transform-3d
    • Springs at the CA level: 11-caspringanimation
    • GPU per-frame: 24-metal-and-calayer, Shaders Roadmap

    The discipline

    When you imagine an interaction, name three things before coding:

    1. What function describes the value? (this folder)
    2. Which driver owns it? (this note)
    3. Which layer renders it? (Core Animation / Metal)

    Get those three and the implementation is almost mechanical.

    10 — Study Method

    The method matters more than the reading list. Four habits.

    1. Desmos first, code second

    Before writing any code, model the interaction as a function in Desmos with sliders. Take its derivative there; watch the tangent move. The interactivity builds intuition that reading can't. This is the most important habit on the page.

    2. Drill the meta-move

    position → derivative → geometric quantity. Pick a curve (wave, circle, spiral, Bézier) and extract its tangent (orientation), normal (perpendicular), and curvature (bend). Do it for five different curves until it's automatic. → 01-modeling-visuals-as-functions, 03-derivative-as-tangent

    3. Reconstruct, math-first

    You already reverse-engineer interactions — keep doing it, but write the function and derive what you need before building. Ask: what curve are the cells on? what's its derivative? what does the gesture change? Write that spec, then code. (This also trains the spec/pseudo-code muscle — turn "scales as it enters" into scale = lerp(0.5, 1, smoothstep(t)) with named constants and stated edge cases.)

    4. Use shaders as the lab

    Your Metal work is applied math — smoothstep, vectors, oscillation are already in there. When you learn a concept, immediately deploy it in a shader. Theory + an existing playground = a fast feedback loop. → 05-math-for-shaders

    A good single rep

    1. See an interaction you like.
    2. Guess the function + driver (09-what-drives-the-value).
    3. Reproduce it in Desmos with sliders.
    4. Derive the tangent/normal/whatever it needs.
    5. Build it (SwiftUI or a shader).
    6. Compare to the original; note what you missed.

    Ten of these reps will move you further than any course.

    Anti-patterns

    • Memorizing formulas without the geometric picture → you'll forget them. Learn the meaning (slope, alignment, turn).
    • Jumping to code before modeling → you end up tuning magic numbers instead of deriving relationships.
    • Reading passively → always have Desmos or a playground open.

    11 — Resources

    Ranked and chosen for the creative-coding / interaction flavor specifically.

    Start here — intuition

    • 3Blue1Brown — "Essence of Calculus" + "Essence of Linear Algebra" (YouTube). The best visual intuition anywhere: derivative-as-slope, dot-product-as-projection. Watch both before anything else. → feeds 03-derivative-as-tangent, 04-vectors-and-linear-algebra
    • The Nature of Code — Daniel Shiffman (natureofcode.com, free; "The Coding Train" on YouTube). Vectors → forces → oscillation → springs → noise, taught by building interactions. The single most on-target end-to-end resource. → 07-oscillations-noise-and-organic-motion

    Go deep — this exact aesthetic

    • Freya Holmér (YouTube) — "Math for Game Devs," "The Continuity of Splines" (tangents/curvature = the carousel math), and her remapping/smoothstep videos. The creator most precisely aimed at your gap. → 05-parametric-curves-and-splines, 06-interpolation-easing-and-springs
    • Inigo Quilez (iquilezles.org + YouTube) — "useful little functions," shaping functions, SDFs, smoothstep. Doubles as shader math. → 05-math-for-shaders
    • The Book of Shaders (thebookofshaders.com) — sine waves, shaping functions, noise; interactive and gentle. Bridges math ↔ your .metal files. → 15-noise-and-patterns

    Understand-through-interaction (gold standard)

    • Bartosz Ciechanowski (ciechanow.ski) — interactive essays. Read "Curves and Surfaces" and "Bézier Curves." Nobody explains it better.
    • Steven Wittens (acko.net) — "Animate Your Way to Glory" for the math of animation/easing.

    Apple / iOS specific

    • "Designing Fluid Interfaces" (WWDC 2018). The talk on gesture-as-function, interruptible, physics-based interactions — exactly your aesthetic. Watch twice.
    • "Explore SwiftUI animation" + "Animate with springs" (WWDC 2023) — the official interpolation/spring model.
    • Wave (Janum Trivedi's spring library, already in your project) — read the source; a masterclass in retargetable springs. → damped-harmonic-oscillator
