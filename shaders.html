    01 — GPU vs CPU: Why Shaders Exist

    The Core Problem

    Your iPhone screen has ~2.7 million pixels (1179 x 2556 on iPhone 15 Pro). At 120fps, that's 324 million pixel calculations per second. A CPU processes things one at a time (or a few at a time with multiple cores). A GPU processes thousands at a time.

    CPU: The Smart Worker

    CPU = 6 very fast, very smart workers
     Each can handle any task
     They work on tasks one by one (mostly)

Task: Color 1 million pixels
CPU approach: Worker 1 colors pixel 1, then pixel 2, then pixel 3...
Time: ~16ms (barely makes 60fps)

    The CPU is great at complex, sequential logic — if/else chains, branching, state machines. But it's terrible at doing the same simple operation millions of times.

    GPU: The Army of Simple Workers

    GPU = 10,000+ simple workers (Apple GPU has thousands of ALUs)
     Each can only do basic math
     But they ALL work simultaneously

Task: Color 1 million pixels
GPU approach: All 10,000 workers color their assigned pixels at the same time
Time: ~0.1ms

    The GPU is terrible at complex branching but incredible at doing the same operation on different data simultaneously. This is called SIMD (Single Instruction, Multiple Data).

    What is a Shader?

    A shader is a small program that runs on the GPU. It's called a "shader" because originally these programs were used to calculate lighting and shading in 3D graphics. Today they do much more.

    You write ONE function.
The GPU runs it on EVERY pixel (or vertex, or data point) simultaneously.

    Think of it like this:

    // CPU approach (Swift)
for y in 0..<screenHeight {
    for x in 0..<screenWidth {
        pixels[x][y] = calculateColor(x, y)  // Sequential: one at a time
    }
}

// GPU approach (Mental model)
// The GPU calls YOUR function for every pixel at the same time
// You write the function for ONE pixel:
func calculateColor(x: Int, y: Int) -> Color {
    return .red  // This runs on ALL pixels simultaneously
}

    Three Types of Shaders

    1. Fragment Shader (Pixel Shader)

    Runs once per pixel. Determines the final color of each pixel on screen.

    Input: pixel position, textures, uniforms
Output: color (RGBA)
Use: Visual effects, post-processing, materials

    This is what you'll use most for iOS visual effects.

    2. Vertex Shader

    Runs once per vertex (corner point of a triangle). Transforms 3D geometry.

    Input: vertex position, normals, texture coordinates
Output: transformed position
Use: 3D transformations, geometry animation, mesh deformation

    3. Compute Shader (Kernel)

    General-purpose GPU computation. Not tied to rendering.

    Input: any data (textures, buffers)
Output: any data
Use: Image processing, physics simulation, machine learning

    Metal Shading Language (MSL)

    Apple's shader language is called Metal Shading Language (MSL). It's based on C++14 with GPU-specific additions.

    C++14 syntax
+ GPU-specific types (float4, half4, texture2d)
+ GPU-specific qualifiers (<span class="xref-missing" title="note not in this site">position</span>, <span class="xref-missing" title="note not in this site">texture(0)</span>)
+ GPU-specific functions (sample, write, threadgroup barriers)
- No dynamic memory allocation
- No recursion
- No virtual functions
- No exceptions

    MSL files have the .metal extension and are compiled by Xcode as part of your build.

    The Rendering Pipeline (Simplified)

    1. Your Swift code sets up data (vertices, textures, parameters)
         |
2. Data is sent to the GPU
         |
3. VERTEX SHADER runs on each vertex
   - Transforms 3D positions to 2D screen positions
         |
4. Rasterizer (hardware, not programmable)
   - Determines which pixels each triangle covers
   - Interpolates vertex data across the triangle
         |
5. FRAGMENT SHADER runs on each pixel
   - Determines the final color
         |
6. Pixels written to framebuffer -> displayed on screen

    For 2D effects (like blur, color grading, distortion), you typically only need a fragment shader operating on a full-screen quad.

    Why This Matters for iOS

    Every visual effect on your iPhone runs through the GPU: - UIVisualEffectView blur? GPU fragment shader. - SwiftUI .shadow()? GPU fragment shader. - Core Animation .cornerRadius? GPU rendering. - Photos app filters? GPU compute shaders. - Every single frame of every animation? GPU.

    When you write Metal shaders, you're writing code at the same level Apple uses for all of iOS.

    Key Concepts to Remember

    Concept Meaning
    Parallel execution Your shader function runs on thousands of pixels simultaneously
    No shared state Each pixel's computation is independent — you can't read what another pixel calculated
    Per-pixel thinking You write logic for ONE pixel; the GPU handles the parallelism
    SIMD Same instruction, different data — the GPU runs your code on different positions/pixels at once
    Framebuffer The final image buffer that gets displayed on screen

    Exercises

    1. Mental model: If you had to turn every pixel on screen to blue, how would you describe it as a shader? (Answer: a function that takes a position and returns blue, regardless of position)

    2. Think parallel: Why can't a blur shader read the result of blurring a neighboring pixel? (Answer: all pixels are computed simultaneously — neighbor hasn't been computed yet when you run)

    3. Count: Your iPhone runs at 120fps with 2.7M pixels. How many fragment shader invocations happen per second? (Answer: 324 million)

    02 — MSL Syntax Basics

    MSL is C++14 with GPU-specific extensions. If you know Swift, you'll find it familiar but with some key differences.

    Swift vs MSL Cheat Sheet

    Swift MSL Notes
    Float float 32-bit float
    Double — GPUs don't use double (too slow)
    Int int 32-bit integer
    UInt uint 32-bit unsigned integer
    Bool bool Boolean
    — half 16-bit float (GPU-optimized, use this!)
    SIMD2<Float> float2 2-component vector
    SIMD3<Float> float3 3-component vector
    SIMD4<Float> float4 4-component vector (RGBA color!)
    func return-type + name C-style function declaration
    let const Immutable
    var plain type Mutable

    Scalar Types

    // Floating point (use these most)
float  x = 1.0;    // 32-bit — standard precision
half   y = 1.0h;   // 16-bit — faster, use when precision isn't critical
// Note the 'h' suffix for half literals

// Integer
int    a = 42;     // 32-bit signed
uint   b = 42u;    // 32-bit unsigned
short  c = 42;     // 16-bit signed
ushort d = 42u;    // 16-bit unsigned

// Boolean
bool flag = true;

    Performance tip: half (16-bit float) is ~2x faster than float (32-bit) on Apple GPUs. Use half for colors and visual effects, float for positions and coordinates.

    Vector Types — The Most Important Thing in MSL

    Vectors are how you work with colors, positions, and directions. They're the bread and butter of shader programming.

    Declaring Vectors

    // 2-component (positions, UVs)
float2 position = float2(100.0, 200.0);
float2 uv = float2(0.5, 0.5);

// 3-component (RGB colors, 3D positions)
float3 color = float3(1.0, 0.0, 0.0);  // Red
float3 position3d = float3(x, y, z);

// 4-component (RGBA colors, homogeneous coordinates)
float4 red = float4(1.0, 0.0, 0.0, 1.0);    // Red, fully opaque
half4  blue = half4(0.0, 0.0, 1.0, 1.0);     // Blue, half precision

// All components same value
float4 white = float4(1.0);  // = float4(1.0, 1.0, 1.0, 1.0)
float4 clear = float4(0.0);  // = float4(0.0, 0.0, 0.0, 0.0)

    Accessing Components — Swizzling

    This is one of the most powerful features. You can access vector components by name and rearrange them freely:

    float4 color = float4(1.0, 0.5, 0.2, 1.0);

// Access individual components
float r = color.r;    // 1.0 (r = first component)
float g = color.g;    // 0.5
float b = color.b;    // 0.2
float a = color.a;    // 1.0

// Same components, different names (both work!)
float x = color.x;    // 1.0 (x = first component)
float y = color.y;    // 0.5
float z = color.z;    // 0.2
float w = color.w;    // 1.0

// SWIZZLING — rearrange components!
float3 rgb = color.rgb;       // float3(1.0, 0.5, 0.2) — drop alpha
float2 rg = color.rg;         // float2(1.0, 0.5)
float3 bgr = color.bgr;       // float3(0.2, 0.5, 1.0) — reversed!
float4 rrrr = color.rrrr;     // float4(1.0, 1.0, 1.0, 1.0) — repeat!
float2 yx = color.yx;         // float2(0.5, 1.0) — swapped!

    Swizzling rules: - You can use .xyzw or .rgba naming (don't mix them) - You can repeat components: .rrr, .xxxx - You can reorder: .bgr, .zyx - The result type matches the number of components: .rg = float2, .rgb = float3

    Vector Math

    All math operations work component-wise:

    float4 a = float4(1.0, 2.0, 3.0, 4.0);
float4 b = float4(2.0, 3.0, 4.0, 5.0);

float4 sum = a + b;        // float4(3.0, 5.0, 7.0, 9.0)
float4 product = a * b;    // float4(2.0, 6.0, 12.0, 20.0)
float4 scaled = a * 2.0;   // float4(2.0, 4.0, 6.0, 8.0)
float4 mixed = a * 0.5 + b * 0.5;  // Average of a and b

    Matrix Types

    float2x2 mat2;   // 2x2 matrix
float3x3 mat3;   // 3x3 matrix
float4x4 mat4;   // 4x4 matrix (standard transform matrix)

// Create identity-like
float4x4 transform = float4x4(1.0);

// Matrix * vector
float4 transformed = transform * position;

    Built-in Functions (The Essentials)

    Math Functions

    // Clamp to range [0, 1]
float clamped = clamp(value, 0.0, 1.0);
// Same as: clamped = min(max(value, 0.0), 1.0)

// Clamp to [0, 1] shorthand
float sat = saturate(value);  // Metal-specific, same as clamp(value, 0.0, 1.0)

// Linear interpolation (LERP)
float mixed = mix(a, b, t);  // = a * (1-t) + b * t
// t=0 returns a, t=1 returns b, t=0.5 returns average
// Works on vectors too:
float4 blended = mix(color1, color2, 0.3);  // 30% of color2

// Smooth interpolation (S-curve)
float smooth = smoothstep(edge0, edge1, x);
// Returns 0 when x < edge0
// Returns 1 when x > edge1
// Smooth curve between edge0 and edge1
// Used EVERYWHERE for soft transitions

// Step function (hard edge)
float hard = step(edge, x);
// Returns 0 when x < edge
// Returns 1 when x >= edge

    Trigonometry

    float s = sin(angle);        // Sine
float c = cos(angle);        // Cosine
float t = tan(angle);        // Tangent
float a = atan2(y, x);       // Angle from coordinates
float2 sc = float2(cos(a), sin(a));  // Point on unit circle

    Distance & Length

    float len = length(vec);              // Magnitude of vector
float dist = distance(point1, point2); // Distance between points
float3 dir = normalize(vec);          // Unit vector (length = 1)
float d = dot(a, b);                  // Dot product
float3 c = cross(a, b);              // Cross product (3D only)

    Useful Shortcuts

    float mn = min(a, b);         // Minimum
float mx = max(a, b);         // Maximum
float ab = abs(value);        // Absolute value
float fr = fract(value);      // Fractional part (value - floor(value))
float fl = floor(value);      // Round down
float cl = ceil(value);       // Round up
float sg = sign(value);       // -1, 0, or 1
float pw = pow(base, exp);    // Power
float sq = sqrt(value);       // Square root
float rs = rsqrt(value);      // 1/sqrt (faster than sqrt on GPU)

    Function Qualifiers — What Makes a Shader a Shader

    These special qualifiers tell Metal which functions are shader entry points:

    // FRAGMENT SHADER — runs per pixel
fragment float4 myFragmentShader(/* params */) {
    return float4(1, 0, 0, 1); // Return a color
}

// VERTEX SHADER — runs per vertex
vertex float4 myVertexShader(/* params */) {
    return float4(0, 0, 0, 1); // Return a position
}

// COMPUTE SHADER (kernel) — runs per thread
kernel void myComputeShader(/* params */) {
    // Write to textures/buffers
}

    Regular helper functions use no qualifier:

    // Regular function — called by shaders, not an entry point
float3 adjustBrightness(float3 color, float amount) {
    return color * amount;
}

    Control Flow

    // If/else (same as C++)
if (uv.x > 0.5) {
    color = float4(1, 0, 0, 1);
} else {
    color = float4(0, 0, 1, 1);
}

// Ternary (prefer this — avoids GPU branching penalty)
color = (uv.x > 0.5) ? red : blue;

// For loops
for (int i = 0; i < 10; i++) {
    sum += values[i];
}

// WARNING: Avoid complex branching!
// GPUs execute BOTH branches and discard one.
// Use math (step, smoothstep, mix) instead of if/else when possible.

    Performance rule: Replace if (condition) { a } else { b } with mix(b, a, step(threshold, value)) when possible. The GPU processes both branches regardless — mix avoids the penalty.

    A Complete Minimal Shader

    #include <metal_stdlib>
using namespace metal;

// This shader turns every pixel red
fragment float4 allRed(float4 position <span class="xref-missing" title="note not in this site">position</span>) {
    return float4(1.0, 0.0, 0.0, 1.0); // RGBA: red, no green, no blue, fully opaque
}

    That's it. That's a complete, valid Metal shader. Everything else builds on top of this.

    Exercises

    1. Vector math: What is float4(1, 0.5, 0, 1).rgb * 2.0? (Answer: float3(2.0, 1.0, 0.0))

    2. Swizzling: If color = float4(0.2, 0.4, 0.6, 0.8), what is color.abgr? (Answer: float4(0.8, 0.6, 0.4, 0.2))

    3. mix: What does mix(float3(1,0,0), float3(0,0,1), 0.5) return? (Answer: float3(0.5, 0.0, 0.5) — purple, the average of red and blue)

    4. smoothstep: What does smoothstep(0.3, 0.7, 0.5) return? (Answer: 0.5 — it's in the exact middle of the transition range)

    03 — Your First Shader: From File to Screen

    Time to write a shader, hook it up to Swift, and see pixels on screen.

    Step 1: Create the Metal File

    In Xcode: File > New > File > Metal File. Name it Shaders.metal.

    #include <metal_stdlib>
using namespace metal;

// A struct to pass data from vertex shader to fragment shader
struct VertexOut {
    float4 position <span class="xref-missing" title="note not in this site">position</span>;  // Required: clip-space position
    float2 uv;                     // Texture coordinate (0 to 1)
};

// Vertex shader: creates a full-screen quad
vertex VertexOut fullscreenVertex(uint vertexID <span class="xref-missing" title="note not in this site">vertex_id</span>) {
    // Full-screen triangle trick: 3 vertices that cover the entire screen
    // No vertex buffer needed!
    VertexOut out;

    // Generate UV coordinates
    out.uv = float2((vertexID << 1) & 2, vertexID & 2);

    // Convert UV (0 to 1) to clip space (-1 to 1), flip Y
    out.position = float4(out.uv * float2(2, -2) + float2(-1, 1), 0, 1);

    return out;
}

// Fragment shader: runs per pixel, returns a color
fragment float4 solidColor(VertexOut in <span class="xref-missing" title="note not in this site">stage_in</span>) {
    // in.uv goes from (0,0) at top-left to (1,1) at bottom-right
    // Return red
    return float4(1.0, 0.0, 0.0, 1.0);
}

    Step 2: The Swift Renderer

    import MetalKit

class ShaderRenderer: NSObject, MTKViewDelegate {
    let device: MTLDevice
    let commandQueue: MTLCommandQueue
    let pipelineState: MTLRenderPipelineState

    init(metalView: MTKView) {
        // 1. Get the GPU
        device = MTLCreateSystemDefaultDevice()!
        metalView.device = device

        // 2. Create command queue (sends work to GPU)
        commandQueue = device.makeCommandQueue()!

        // 3. Load shader functions
        let library = device.makeDefaultLibrary()!
        let vertexFunc = library.makeFunction(name: "fullscreenVertex")!
        let fragmentFunc = library.makeFunction(name: "solidColor")!

        // 4. Create render pipeline
        let descriptor = MTLRenderPipelineDescriptor()
        descriptor.vertexFunction = vertexFunc
        descriptor.fragmentFunction = fragmentFunc
        descriptor.colorAttachments[0].pixelFormat = metalView.colorPixelFormat

        pipelineState = try! device.makeRenderPipelineState(descriptor: descriptor)

        super.init()
        metalView.delegate = self
    }

    // Called every frame
    func draw(in view: MTKView) {
        guard let drawable = view.currentDrawable,
              let renderPassDescriptor = view.currentRenderPassDescriptor else { return }

        let commandBuffer = commandQueue.makeCommandBuffer()!
        let encoder = commandBuffer.makeRenderCommandEncoder(descriptor: renderPassDescriptor)!

        encoder.setRenderPipelineState(pipelineState)
        encoder.drawPrimitives(type: .triangle, vertexStart: 0, vertexCount: 3)

        encoder.endEncoding()
        commandBuffer.present(drawable)
        commandBuffer.commit()
    }

    func mtkView(_ view: MTKView, drawableSizeWillChange size: CGSize) {}
}

    Step 3: Hook It Up

    UIKit

    import UIKit
import MetalKit

class ViewController: UIViewController {
    var renderer: ShaderRenderer!

    override func viewDidLoad() {
        super.viewDidLoad()

        let metalView = MTKView(frame: view.bounds)
        metalView.autoresizingMask = [.flexibleWidth, .flexibleHeight]
        view.addSubview(metalView)

        renderer = ShaderRenderer(metalView: metalView)
    }
}

import MetalKit

struct MetalView: UIViewRepresentable {

    }
}

    Step 5: Make It Interesting

    Now modify the fragment shader to use the UV coordinates:

    Gradient

    fragment float4 gradient(VertexOut in <span class="xref-missing" title="note not in this site">stage_in</span>) {
    // in.uv.x goes 0 (left) to 1 (right)
    // in.uv.y goes 0 (top) to 1 (bottom)

    // Horizontal gradient: black on left, red on right
    return float4(in.uv.x, 0.0, 0.0, 1.0);
}

    Two-Color Gradient

    fragment float4 twoColorGradient(VertexOut in <span class="xref-missing" title="note not in this site">stage_in</span>) {
    float3 topColor = float3(0.1, 0.1, 0.4);     // Dark blue
    float3 bottomColor = float3(1.0, 0.5, 0.2);   // Orange

    // Mix based on vertical position
    float3 color = mix(topColor, bottomColor, in.uv.y);

    return float4(color, 1.0);
}

    Horizontal Split

    fragment float4 horizontalSplit(VertexOut in <span class="xref-missing" title="note not in this site">stage_in</span>) {
    // step returns 0 when uv.x < 0.5, 1 when uv.x >= 0.5
    float side = step(0.5, in.uv.x);

    float3 left = float3(1.0, 0.0, 0.0);   // Red
    float3 right = float3(0.0, 0.0, 1.0);  // Blue

    float3 color = mix(left, right, side);
    return float4(color, 1.0);
}

    Circle

    fragment float4 circle(VertexOut in <span class="xref-missing" title="note not in this site">stage_in</span>) {
    // Center of screen
    float2 center = float2(0.5, 0.5);

    // Distance from center
    float dist = distance(in.uv, center);

    // Sharp circle at radius 0.3
    // float mask = step(dist, 0.3);

    // Soft circle (smooth edge)
    float mask = 1.0 - smoothstep(0.28, 0.32, dist);

    float3 circleColor = float3(0.0, 0.8, 1.0);  // Cyan
    float3 bgColor = float3(0.05, 0.05, 0.1);     // Dark

    float3 color = mix(bgColor, circleColor, mask);
    return float4(color, 1.0);
}

    Understanding the Full-Screen Triangle Trick

    The vertex shader uses a clever trick to cover the screen with just 3 vertices (a single triangle) instead of 6 (two triangles forming a quad):

    vertexID=0: uv=(0,0) -> position=(-1, 1)   Top-left
vertexID=1: uv=(2,0) -> position=( 3, 1)   Far right (off-screen)
vertexID=2: uv=(0,2) -> position=(-1,-3)   Far bottom (off-screen)

     0-----------1 (off screen right)
     |  /screen/ |
     | /      /  |
     |/______/   |
     2           |
     (off screen bottom)

    The GPU clips to the screen bounds, so the oversized triangle just fills the viewport. This is more efficient than a quad because it avoids the diagonal edge where two triangles would meet.

    04 — Coordinates and Color

    Understanding how position and color work in shader space is essential for everything that follows.

    Coordinate Systems

    UV Space (Normalized)

    (0,0)───────────────(1,0)
  │                    │
  │    UV SPACE        │
  │                    │
  │       (0.5,0.5)    │
  │         center     │
  │                    │
(0,1)───────────────(1,1)
    • Range: 0.0 to 1.0
    • Origin: top-left
    • This is what your fragment shader receives in VertexOut.uv
    • Resolution-independent — same coordinates whether on iPhone SE or iPad Pro

    Pixel Space

    (0,0)────────────────(1179,0)
  │                       │
  │    PIXEL SPACE        │
  │    (actual pixels)    │
  │                       │
(0,2556)──────────(1179,2556)
    • <span class="xref-missing" title="note not in this site">position</span> gives you pixel coordinates
    • Resolution-dependent
    • Convert between them:
    // Pixel to UV
float2 uv = position.xy / viewportSize;

// UV to pixel
float2 pixel = uv * viewportSize;

    Centered Coordinate System

    Often you want (0,0) at the center with proper aspect ratio:

    fragment float4 centeredCoords(VertexOut in <span class="xref-missing" title="note not in this site">stage_in</span>,
                                constant float2 &resolution <span class="xref-missing" title="note not in this site">buffer(0)</span>) {
    // Convert UV to centered coordinates
    float2 uv = in.uv;
    uv = uv * 2.0 - 1.0;          // Remap from [0,1] to [-1,1]
    uv.x *= resolution.x / resolution.y;  // Fix aspect ratio

    // Now (0,0) is center, and circles look like circles (not ovals)

    float dist = length(uv);  // Distance from center
    float circle = 1.0 - smoothstep(0.4, 0.42, dist);

    return float4(float3(circle), 1.0);
}

    Why fix aspect ratio? Without it, distance calculations treat horizontal and vertical pixels as equal units. Since screens are taller than wide, a "circle" becomes an oval. Multiplying x by width/height corrects this.

    Color in Shaders

    RGBA

    Colors in Metal are float4 (or half4) with components R, G, B, A:

    float4 red     = float4(1, 0, 0, 1);     // Full red, opaque
float4 green   = float4(0, 1, 0, 1);     // Full green, opaque
float4 blue    = float4(0, 0, 1, 1);     // Full blue, opaque
float4 white   = float4(1, 1, 1, 1);     // All channels max
float4 black   = float4(0, 0, 0, 1);     // All channels zero
float4 clear   = float4(0, 0, 0, 0);     // Fully transparent
float4 gray50  = float4(0.5, 0.5, 0.5, 1);  // 50% gray

// Semi-transparent
float4 transRed = float4(1, 0, 0, 0.5);  // Red at 50% opacity

    Color Mixing

    // Linear interpolation between two colors
float4 sunrise = mix(
    float4(0.1, 0.0, 0.2, 1.0),  // Deep purple
    float4(1.0, 0.6, 0.1, 1.0),  // Warm orange
    in.uv.y                        // Blend by vertical position
);

// Multiply (darkens — used for shadows, tinting)
float4 tinted = baseColor * float4(1.0, 0.9, 0.8, 1.0);  // Warm tint

// Screen (brightens — used for highlights)
float4 screened = 1.0 - (1.0 - baseColor) * (1.0 - highlightColor);

// Additive (pure light addition — used for glows)

    Common Color Operations

    // Brightness
float3 brighter = color.rgb * 1.5;     // 50% brighter
float3 darker = color.rgb * 0.5;       // 50% darker

// Grayscale (perceptual luminance weights)
float luminance = dot(color.rgb, float3(0.2126, 0.7152, 0.0722));
float3 gray = float3(luminance);

// Contrast
float3 contrasted = (color.rgb - 0.5) * contrastAmount + 0.5;

// Saturation

// saturationAmount: 0 = grayscale, 1 = original, >1 = oversaturated

// Invert

// Hue shift (rotate color components)
// Simple approximation:

    Premultiplied Alpha

    Metal (and Core Animation) use premultiplied alpha by default. This means the RGB values are already multiplied by alpha:

    // Straight alpha (what you might expect):
// RGBA = (1.0, 0.0, 0.0, 0.5) means "red at 50% opacity"

// Premultiplied alpha (what Metal uses):
// RGBA = (0.5, 0.0, 0.0, 0.5) means "red at 50% opacity"
// RGB is pre-multiplied by A: (1.0 * 0.5, 0.0 * 0.5, 0.0 * 0.5, 0.5)

// Convert straight to premultiplied:
float4 premultiplied = float4(color.rgb * color.a, color.a);

// Why? Premultiplied blending is a single operation:
// result = src + dst * (1 - src.a)
// vs straight alpha which needs:
// result = src * src.a + dst * (1 - src.a)  -- two multiplies

    Practical Gradient Recipes

    Radial Gradient

    fragment float4 radialGradient(VertexOut in <span class="xref-missing" title="note not in this site">stage_in</span>) {
    float2 center = float2(0.5, 0.5);
    float dist = distance(in.uv, center);

    // Normalize distance (0 at center, 1 at edge)
    float t = saturate(dist / 0.5);

    float3 innerColor = float3(1.0, 0.8, 0.0);  // Gold
    float3 outerColor = float3(0.1, 0.0, 0.2);  // Deep purple

    return float4(mix(innerColor, outerColor, t), 1.0);
}

    Angular Gradient (Conic)

    fragment float4 angularGradient(VertexOut in <span class="xref-missing" title="note not in this site">stage_in</span>) {
    float2 centered = in.uv - 0.5;

    // atan2 returns angle in radians (-pi to pi)
    float angle = atan2(centered.y, centered.x);

    // Normalize to 0-1
    float t = (angle / (2.0 * M_PI_F)) + 0.5;

    // Rainbow
    float3 color = float3(
        sin(t * 6.28) * 0.5 + 0.5,
        sin(t * 6.28 + 2.09) * 0.5 + 0.5,
        sin(t * 6.28 + 4.19) * 0.5 + 0.5
    );

    return float4(color, 1.0);
}

    Multi-Stop Gradient

    fragment float4 multiStopGradient(VertexOut in <span class="xref-missing" title="note not in this site">stage_in</span>) {
    float t = in.uv.y;  // Vertical position

    float3 color;
    if (t < 0.33) {
        color = mix(float3(1,0,0), float3(0,1,0), t / 0.33);         // Red -> Green
    } else if (t < 0.66) {
        color = mix(float3(0,1,0), float3(0,0,1), (t - 0.33) / 0.33); // Green -> Blue
    } else {
        color = mix(float3(0,0,1), float3(1,0,0), (t - 0.66) / 0.33); // Blue -> Red
    }

    return float4(color, 1.0);
}

    Signed Distance Fields (SDF) — Drawing Shapes

    SDFs are a powerful technique where you define shapes by their distance function. The distance is negative inside, zero at the edge, and positive outside.

    Circle SDF

    float sdCircle(float2 point, float2 center, float radius) {
    return distance(point, center) - radius;
    // Negative = inside the circle
    // Zero = on the edge
    // Positive = outside the circle
}

fragment float4 sdfCircle(VertexOut in <span class="xref-missing" title="note not in this site">stage_in</span>) {
    float2 uv = in.uv * 2.0 - 1.0;  // Center the coordinates

    float d = sdCircle(uv, float2(0), 0.5);

    // Hard edge
    // float shape = step(d, 0.0);

    // Soft edge (anti-aliased)
    float shape = 1.0 - smoothstep(-0.01, 0.01, d);

    return float4(float3(shape), 1.0);
}

    Rounded Rectangle SDF

    float sdRoundedRect(float2 point, float2 halfSize, float radius) {
    float2 d = abs(point) - halfSize + radius;
    return length(max(d, 0.0)) + min(max(d.x, d.y), 0.0) - radius;
}

fragment float4 sdfRoundedRect(VertexOut in <span class="xref-missing" title="note not in this site">stage_in</span>) {
    float2 uv = in.uv * 2.0 - 1.0;

    float d = sdRoundedRect(uv, float2(0.6, 0.3), 0.1);
    float shape = 1.0 - smoothstep(-0.005, 0.005, d);

    // Bonus: outline
    float outline = 1.0 - smoothstep(0.0, 0.01, abs(d) - 0.02);

    float3 color = mix(float3(0.1), float3(0.0, 0.5, 1.0), shape);
    color = mix(color, float3(1.0), outline);

    return float4(color, 1.0);
}

    Combining Shapes

    // Union (OR) — combine two shapes
float sdUnion(float d1, float d2) { return min(d1, d2); }

// Intersection (AND) — where both shapes overlap
float sdIntersect(float d1, float d2) { return max(d1, d2); }

// Subtraction — cut one shape from another
float sdSubtract(float d1, float d2) { return max(d1, -d2); }

// Smooth union — blend shapes together
float sdSmoothUnion(float d1, float d2, float k) {
    float h = saturate(0.5 + 0.5 * (d2 - d1) / k);
    return mix(d2, d1, h) - k * h * (1.0 - h);
}

    05 — Essential Math for Shaders

    You don't need a math degree. You need ~10 functions and the intuition for when to use them. This lesson gives you both.

    The Big Five: Functions You'll Use Every Day

    1. mix(a, b, t) — Linear Interpolation

    The most important function in shader programming. Blends between two values.

    mix(a, b, t) = a * (1.0 - t) + b * t

// t = 0.0  ->  returns a
// t = 0.5  ->  returns (a + b) / 2  (average)
// t = 1.0  ->  returns b
// t = 0.25 ->  returns 75% a + 25% b

    // Blend colors
float3 color = mix(red, blue, 0.5);        // Purple

// Fade based on distance
float alpha = mix(1.0, 0.0, distance);     // Fade out with distance

// Animate between states
float3 pos = mix(startPos, endPos, progress);  // Move from A to B

    Works on all types: float, float2, float3, float4. Each component is interpolated independently.

    2. step(edge, x) — Hard Threshold

    step(edge, x) = (x >= edge) ? 1.0 : 0.0

    // Split screen in half
float mask = step(0.5, uv.x);          // 0 on left, 1 on right

// Threshold a gradient
float bw = step(0.3, grayscale);        // Black below 0.3, white above

// Create a border at specific distance
float border = step(0.49, uv.x) * step(uv.x, 0.51);  // Thin vertical line

    3. smoothstep(edge0, edge1, x) — Smooth Threshold

    smoothstep(edge0, edge1, x):
  x < edge0  ->  0.0
  x > edge1  ->  1.0
  between    ->  smooth S-curve (Hermite interpolation)

    This is step but with a smooth transition instead of a sharp cut. The transition zone is between edge0 and edge1.

    // Soft circle edge
float circle = 1.0 - smoothstep(0.28, 0.32, dist);
// dist < 0.28: fully inside (1.0)
// dist > 0.32: fully outside (0.0)
// 0.28 to 0.32: smooth gradient (anti-aliased edge)

// Fog effect
float fog = smoothstep(0.0, 100.0, depth);

// Smooth color band
float band = smoothstep(0.3, 0.35, uv.y) - smoothstep(0.65, 0.7, uv.y);

    Tip: The width between edge0 and edge1 controls the softness. Narrow = sharper, wide = softer.

    4. clamp(x, minVal, maxVal) / saturate(x)

    clamp(x, 0.0, 1.0)  // Forces x into [0, 1] range
saturate(x)           // Same thing, Metal shorthand (faster on Apple GPUs)

    // Prevent color overflow
float3 color = saturate(brightColor);  // Clamps each component to [0, 1]

// Limit a parameter
float radius = clamp(userRadius, 1.0, 50.0);

    5. fract(x) — Fractional Part

    fract(x) = x - floor(x)
// Always returns a value between 0.0 and 1.0

fract(0.7)  = 0.7
fract(1.7)  = 0.7
fract(2.7)  = 0.7
fract(-0.3) = 0.7

    This creates repeating patterns:

    // Repeating stripes
float stripes = step(0.5, fract(uv.x * 10.0));  // 10 black/white stripes

// Repeating tiles
float2 tileUV = fract(uv * 4.0);  // 4x4 grid of repeated UV spaces

// Sawtooth wave (goes 0->1, 0->1, 0->1...)
float saw = fract(time);

// Triangle wave (goes 0->1->0->1->0...)
float tri = abs(fract(time) * 2.0 - 1.0);

    Trigonometry: Circles, Waves, Rotation

    sin and cos

    // Sine wave: oscillates between -1 and 1
float wave = sin(time);           // Oscillates over time
float wave01 = sin(time) * 0.5 + 0.5;  // Remap to [0, 1]

// Cosine: same as sine but shifted by 90 degrees
// sin and cos together trace a circle:
float2 circlePoint = float2(cos(angle), sin(angle)) * radius;

    Creating Waves

    // Horizontal wave
float wave = sin(uv.x * 20.0) * 0.02;  // 20 waves, 0.02 amplitude
float2 distortedUV = float2(uv.x, uv.y + wave);

// Animated wave
float wave = sin(uv.x * 10.0 + time * 3.0) * 0.05;

// Compound wave (more organic)
float wave = sin(uv.x * 10.0 + time) * 0.03
           + sin(uv.x * 23.0 + time * 1.5) * 0.015
           + sin(uv.x * 7.0 - time * 0.8) * 0.02;

    Rotation

    // Rotate a 2D point around origin
float2 rotate(float2 point, float angle) {
    float c = cos(angle);
    float s = sin(angle);
    return float2(
        point.x * c - point.y * s,
        point.x * s + point.y * c
    );
}

// Usage: rotate UV coordinates
float2 centered = uv - 0.5;         // Move origin to center
float2 rotated = rotate(centered, time);  // Rotate over time
float2 finalUV = rotated + 0.5;     // Move origin back

    atan2 — Angle From Position

    float angle = atan2(y, x);
// Returns angle in radians from -pi to pi
// (0,1) -> pi/2 (straight up)
// (1,0) -> 0 (right)
// (0,-1) -> -pi/2 (down)
// (-1,0) -> pi (left)

// Normalize to [0, 1]:
float normalizedAngle = (atan2(y, x) / (2.0 * M_PI_F)) + 0.5;

    Distance Functions

    // Euclidean distance (standard "as the crow flies")
float d = distance(a, b);  // = length(a - b) = sqrt((a.x-b.x)^2 + (a.y-b.y)^2)

// Length of a vector
float len = length(vec);

// Squared length (FASTER — no sqrt, use for comparisons)
float lenSq = dot(vec, vec);  // = vec.x*vec.x + vec.y*vec.y
// Use when you only need to compare distances:
if (dot(vec, vec) < radius * radius) { ... }  // Faster than length(vec) < radius

// Manhattan distance (grid-based, "taxi cab")
float manhattan = abs(a.x - b.x) + abs(a.y - b.y);
// Creates diamond shapes instead of circles

// Chebyshev distance (chessboard)
float chebyshev = max(abs(a.x - b.x), abs(a.y - b.y));
// Creates square shapes

    Mapping and Remapping

    Remap a Value Between Ranges

    // Remap from [inMin, inMax] to [outMin, outMax]
float remap(float value, float inMin, float inMax, float outMin, float outMax) {
    float t = (value - inMin) / (inMax - inMin);  // Normalize to [0, 1]
    return mix(outMin, outMax, t);                  // Map to output range
}

// Examples:
float uiValue = remap(sin(time), -1.0, 1.0, 0.0, 255.0);  // Sine -> 0-255
float screenX = remap(normalizedX, 0.0, 1.0, -1.0, 1.0);   // UV -> clip space

    Applying Easing (Curved Interpolation)

    // Linear (no easing)
float linear = t;

// Ease in (slow start)
float easeIn = t * t;               // Quadratic
float easeInCubic = t * t * t;      // Cubic (stronger)

// Ease out (slow end)
float easeOut = 1.0 - (1.0 - t) * (1.0 - t);           // Quadratic
float easeOutCubic = 1.0 - pow(1.0 - t, 3.0);           // Cubic

// Ease in-out (slow start and end)
float easeInOut = smoothstep(0.0, 1.0, t);  // smoothstep IS ease in-out!

// Bounce (overshoots then settles)
float bounce = 1.0 - pow(1.0 - t, 3.0) * cos(t * 10.0 * M_PI_F);

// Elastic
float elastic = pow(2.0, -10.0 * t) * sin((t - 0.075) * (2.0 * M_PI_F) / 0.3) + 1.0;

    Modular Arithmetic for Patterns

    // Repeat space (tiles)
float2 tiledUV = fract(uv * float2(4, 4));  // 4x4 tiles

// Repeat with offset (brick pattern)
float2 brickUV = uv * float2(4, 8);
brickUV.x += step(1.0, fmod(brickUV.y, 2.0)) * 0.5;  // Offset every other row
brickUV = fract(brickUV);

// Mirror repeat
float mirrorUV = abs(fract(uv.x * 2.0) * 2.0 - 1.0);  // Ping-pong 0->1->0

    Dot Product — Understanding It Intuitively

    dot(a, b) = a.x*b.x + a.y*b.y + a.z*b.z

    What does it actually measure? - Parallel vectors (same direction): dot = positive (maximum when both are unit vectors: 1.0) - Perpendicular vectors: dot = 0 - Opposite vectors: dot = negative (minimum: -1.0)

    // Lighting: how much does a surface face the light?
float brightness = max(dot(surfaceNormal, lightDirection), 0.0);
// 1.0 = directly facing light (brightest)
// 0.0 = perpendicular or facing away (darkest)

// Luminance: weighted sum of RGB (dot product with weight vector)
float luma = dot(color.rgb, float3(0.2126, 0.7152, 0.0722));

// Projection: how far along a direction?
float projection = dot(point, normalize(direction));

    <span class="xref-missing" title="note not in this site">stage_in</span> — Interpolated Vertex Data

    The rasterizer takes the vertex shader's output and interpolates it across the triangle for each pixel. This is how UV coordinates smoothly vary across the screen.

    struct VertexOut {
    float4 position <span class="xref-missing" title="note not in this site">position</span>;  // REQUIRED: pixel position
    float2 uv;                     // UV coordinates (0 to 1)
    float3 normal;                 // Surface normal (for 3D)
    float4 color;                  // Per-vertex color
};

// between the 3 vertices of each triangle.

    // .w = 1/w (for perspective correction)
    return float4(1, 0, 0, 1);

    Sampler options:

    Option Values Meaning
    coord normalized, pixel UV range: [0,1] vs [0,width]
    address clamp_to_edge, repeat, mirrored_repeat, clamp_to_zero Out-of-bounds behavior
    filter nearest, linear Pixel interpolation (sharp vs smooth)
    mip_filter none, nearest, linear Mipmap filtering

    Return Type: float4 vs half4

    // Standard precision (32-bit per channel)

    return float4(1, 0, 0, 1);
}

    return half4(1, 0, 0, 1);
}

    Use half4 when: - Your render target is 8-bit or 10-bit (most of the time) - You don't need extreme color precision - Performance matters (always)

    Use float4 when: - Rendering to a float32 texture (HDR, compute intermediate) - Doing precision-sensitive calculations

    Real-World Fragment Shader Patterns

    Pattern 1: Post-Processing Pipeline

    Process an existing rendered image:

    fragment half4 postProcess(
    VertexOut in <span class="xref-missing" title="note not in this site">stage_in</span>,
    texture2d<half> sceneTexture <span class="xref-missing" title="note not in this site">texture(0)</span>)
{
    constexpr sampler s(filter::linear);
    half4 color = sceneTexture.sample(s, in.uv);

    // Vignette
    float2 centered = in.uv - 0.5;
    float vignette = 1.0 - dot(centered, centered) * 1.5;
    color.rgb *= vignette;

    // Slight warm tint
    color.r *= 1.05h;
    color.b *= 0.95h;

    return color;
}

    Pattern 2: Procedural Effect (No Input Texture)

    fragment half4 plasma(
    VertexOut in <span class="xref-missing" title="note not in this site">stage_in</span>,
    constant float &time <span class="xref-missing" title="note not in this site">buffer(0)</span>)
{
    float2 uv = in.uv * 10.0;

    float v1 = sin(uv.x + time);
    float v2 = sin(uv.y + time * 0.7);
    float v3 = sin(uv.x + uv.y + time * 0.5);
    float v4 = sin(length(uv - 5.0) + time);

    float v = (v1 + v2 + v3 + v4) * 0.25;

    half3 color = half3(
        sin(v * M_PI_F) * 0.5h + 0.5h,
        sin(v * M_PI_F + 2.094h) * 0.5h + 0.5h,
        sin(v * M_PI_F + 4.189h) * 0.5h + 0.5h
    );

    return half4(color, 1.0h);
}

    Pattern 3: Multi-Texture Compositing

    fragment half4 composite(
    VertexOut in <span class="xref-missing" title="note not in this site">stage_in</span>,
    texture2d<half> background <span class="xref-missing" title="note not in this site">texture(0)</span>,
    texture2d<half> foreground <span class="xref-missing" title="note not in this site">texture(1)</span>,
    texture2d<half> mask <span class="xref-missing" title="note not in this site">texture(2)</span>,
    constant float &blendAmount <span class="xref-missing" title="note not in this site">buffer(0)</span>)
{
    constexpr sampler s(filter::linear);

    half4 bg = background.sample(s, in.uv);
    half4 fg = foreground.sample(s, in.uv);
    half maskValue = mask.sample(s, in.uv).r;

    // Blend using mask and blend amount
    half4 result = mix(bg, fg, maskValue * half(blendAmount));

    return result;
}

    Pattern 4: Touch-Interactive Effect

    struct TouchParams {
    float2 touchPoint;   // Normalized touch position
    float radius;        // Effect radius
    float time;          // Time since touch
    float2 resolution;
};

fragment half4 touchRipple(
    VertexOut in <span class="xref-missing" title="note not in this site">stage_in</span>,
    texture2d<half> tex <span class="xref-missing" title="note not in this site">texture(0)</span>,
    constant TouchParams &touch <span class="xref-missing" title="note not in this site">buffer(0)</span>)
{
    constexpr sampler s(filter::linear);

    float2 uv = in.uv;
    float dist = distance(uv, touch.touchPoint);

    // Ripple distortion
    float ripple = sin(dist * 40.0 - touch.time * 8.0);
    float falloff = exp(-dist * 5.0) * exp(-touch.time * 2.0);
    float2 offset = normalize(uv - touch.touchPoint) * ripple * falloff * 0.02;

    half4 color = tex.sample(s, uv + offset);
    return color;
}

    07 — Vertex Shaders

    Vertex shaders transform geometry — they determine WHERE pixels end up on screen. While fragment shaders decide color, vertex shaders decide position.

    What Vertex Shaders Do

    3D World                    2D Screen
============               ============
    *  * (vertices)   -->     ** (projected)
   / \/ \                   /  \
  *----*                   *----*

Vertex shader transforms each * from 3D world space to 2D screen space

    The Pipeline

    Vertex Data (positions, normals, UVs)
        |
  VERTEX SHADER (runs per vertex)
  - Transform position (model -> world -> camera -> clip space)
  - Pass data to fragment shader
        |
  Rasterizer (hardware)
  - Determines which pixels each triangle covers
  - Interpolates vertex outputs across triangle surface
        |
  FRAGMENT SHADER (runs per pixel)
  - Receives interpolated vertex output
  - Returns final pixel color

    Basic Vertex Shader

    #include <metal_stdlib>
using namespace metal;

struct VertexIn {
    float3 position <span class="xref-missing" title="note not in this site">attribute(0)</span>;   // 3D position
    float2 texCoord <span class="xref-missing" title="note not in this site">attribute(1)</span>;   // Texture coordinate
    float3 normal   <span class="xref-missing" title="note not in this site">attribute(2)</span>;   // Surface normal
};

struct VertexOut {
    float4 position <span class="xref-missing" title="note not in this site">position</span>;       // REQUIRED: clip-space position
    float2 texCoord;                    // Passed to fragment shader
    float3 normal;                      // Passed to fragment shader
    float3 worldPosition;              // For lighting calculations
};

struct Uniforms {
    float4x4 modelMatrix;       // Object -> World
    float4x4 viewMatrix;        // World -> Camera
    float4x4 projectionMatrix;  // Camera -> Clip (perspective)
};

vertex VertexOut basicVertex(
    VertexIn in <span class="xref-missing" title="note not in this site">stage_in</span>,
    constant Uniforms &uniforms <span class="xref-missing" title="note not in this site">buffer(1)</span>)
{
    VertexOut out;

    // Transform position through the full matrix chain
    float4 worldPos = uniforms.modelMatrix * float4(in.position, 1.0);
    out.worldPosition = worldPos.xyz;
    out.position = uniforms.projectionMatrix * uniforms.viewMatrix * worldPos;

    // Transform normal (use inverse transpose of model matrix for correct normals)
    out.normal = (uniforms.modelMatrix * float4(in.normal, 0.0)).xyz;

    // Pass through texture coordinates
    out.texCoord = in.texCoord;

    return out;
}

    Coordinate Spaces

    MODEL SPACE (Object Space)
  Vertex positions relative to object center
  e.g., a cube from (-1,-1,-1) to (1,1,1)
         |
         | Model Matrix (translate, rotate, scale)
         v
WORLD SPACE
  Position in the scene
  e.g., cube at position (5, 0, -3)
         |
         | View Matrix (camera transform)
         v
VIEW SPACE (Camera Space / Eye Space)
  Position relative to camera
  Camera is at origin, looking down -Z
         |
         | Projection Matrix (perspective or orthographic)
         v
CLIP SPACE
  Normalized coordinates for GPU clipping
  X, Y in [-1, 1], Z in [0, 1]
         |
         | Perspective Divide (hardware, automatic)
         v
NDC (Normalized Device Coordinates)
  After dividing by W component
         |
         | Viewport Transform (hardware, automatic)
         v
SCREEN SPACE (Pixel Coordinates)
  Final pixel positions on screen

    Vertex Shader Animations

    Wobble / Wave Deformation

    vertex VertexOut wobbleVertex(
    VertexIn in <span class="xref-missing" title="note not in this site">stage_in</span>,
    constant Uniforms &uniforms <span class="xref-missing" title="note not in this site">buffer(1)</span>,
    constant float &time <span class="xref-missing" title="note not in this site">buffer(2)</span>)
{
    VertexOut out;

    // Displace Y position based on X position and time
    float3 pos = in.position;
    pos.y += sin(pos.x * 5.0 + time * 3.0) * 0.1;
    pos.y += cos(pos.z * 3.0 + time * 2.0) * 0.05;

    float4 worldPos = uniforms.modelMatrix * float4(pos, 1.0);
    out.position = uniforms.projectionMatrix * uniforms.viewMatrix * worldPos;
    out.texCoord = in.texCoord;
    out.normal = in.normal;
    out.worldPosition = worldPos.xyz;

    return out;
}

    Explode Effect

    vertex VertexOut explodeVertex(
    VertexIn in <span class="xref-missing" title="note not in this site">stage_in</span>,
    constant Uniforms &uniforms <span class="xref-missing" title="note not in this site">buffer(1)</span>,
    constant float &progress <span class="xref-missing" title="note not in this site">buffer(2)</span>)  // 0 = normal, 1 = fully exploded
{
    VertexOut out;

    // Push vertices outward along their normal
    float3 pos = in.position + in.normal * progress * 2.0;

    // Add some randomness based on position
    float hash = fract(sin(dot(in.position.xy, float2(12.9898, 78.233))) * 43758.5453);
    pos += in.normal * hash * progress * 0.5;

    float4 worldPos = uniforms.modelMatrix * float4(pos, 1.0);
    out.position = uniforms.projectionMatrix * uniforms.viewMatrix * worldPos;
    out.texCoord = in.texCoord;
    out.normal = in.normal;
    out.worldPosition = worldPos.xyz;

    return out;
}

    Scale Bounce

    vertex VertexOut bounceVertex(
    VertexIn in <span class="xref-missing" title="note not in this site">stage_in</span>,
    constant Uniforms &uniforms <span class="xref-missing" title="note not in this site">buffer(1)</span>,
    constant float &time <span class="xref-missing" title="note not in this site">buffer(2)</span>)
{
    VertexOut out;

    // Bounce scale using damped spring formula
    float t = fract(time * 0.5);
    float bounce = 1.0 + exp(-t * 5.0) * sin(t * 20.0) * 0.3;
    float3 pos = in.position * bounce;

    float4 worldPos = uniforms.modelMatrix * float4(pos, 1.0);
    out.position = uniforms.projectionMatrix * uniforms.viewMatrix * worldPos;
    out.texCoord = in.texCoord;
    out.normal = in.normal;
    out.worldPosition = worldPos.xyz;

    return out;
}

    Full-Screen Quad (For Post-Processing)

vertex VertexOut fullscreenVertex(uint vertexID <span class="xref-missing" title="note not in this site">vertex_id</span>) {

    out.texCoord = float2((vertexID << 1) & 2, vertexID & 2);
    out.position = float4(out.texCoord * float2(2, -2) + float2(-1, 1), 0, 1);
    return out;
}

    08 — Compute Shaders (Kernels)

    Compute shaders are the most flexible shader type. They're not tied to the rendering pipeline — they're general-purpose GPU programs. Apple uses them for image processing, blur effects, physics simulation, and machine learning.

    Compute vs Render Shaders

    RENDER PIPELINE:
  Vertex Shader -> Rasterizer -> Fragment Shader -> Framebuffer
  Fixed flow, designed for drawing triangles/pixels

COMPUTE PIPELINE:
  Kernel Function -> Read/Write Textures/Buffers
  Free-form, you control everything

    When to use compute shaders: - Image processing (blur, filters, effects) - When you need to READ and WRITE to the same texture - When you need to coordinate between threads (threadgroup memory) - Data processing that doesn't map to rendering (physics, sorting)

    Basic Compute Shader

    #include <metal_stdlib>
using namespace metal;

// kernel = compute shader entry point
kernel void brighten(
    texture2d<float, access::read>  inTexture  <span class="xref-missing" title="note not in this site">texture(0)</span>,   // Read-only input
    texture2d<float, access::write> outTexture <span class="xref-missing" title="note not in this site">texture(1)</span>,   // Write-only output
    uint2 gid <span class="xref-missing" title="note not in this site">thread_position_in_grid</span>)                        // Which pixel am I?
{
    // Bounds check — don't write outside the texture
    if (gid.x >= outTexture.get_width() || gid.y >= outTexture.get_height()) {
        return;
    }

    // Read pixel at this position
    float4 color = inTexture.read(gid);

    // Process it
    color.rgb *= 1.5;  // 50% brighter
    color.rgb = saturate(color.rgb);  // Clamp to [0, 1]

    // Write result
    outTexture.write(color, gid);
}

│ │0,0│ │1,0│ │2,0│... │
│ └───┘ └───┘ └───┘    │
│ ┌───┐ ┌───┐ ┌───┐    │
│ │TG │ │TG │ │TG │    │
│ │0,1│ │1,1│ │2,1│... │
│ └───┘ └───┘ └───┘    │
│ ...                    │

┌──────────────┐

    Swift Side: Dispatching Compute Work

    class ComputeProcessor {
    let device: MTLDevice
    let commandQueue: MTLCommandQueue
    let pipeline: MTLComputePipelineState

    init() {
        device = MTLCreateSystemDefaultDevice()!
        commandQueue = device.makeCommandQueue()!

        let library = device.makeDefaultLibrary()!
        let function = library.makeFunction(name: "brighten")!
        pipeline = try! device.makeComputePipelineState(function: function)
    }

    func process(input: MTLTexture, output: MTLTexture) {
        let commandBuffer = commandQueue.makeCommandBuffer()!
        let encoder = commandBuffer.makeComputeCommandEncoder()!

        encoder.setComputePipelineState(pipeline)
        encoder.setTexture(input, index: 0)
        encoder.setTexture(output, index: 1)

        // Thread group size (typically 8x8 or 16x16)
        let threadGroupSize = MTLSize(width: 16, height: 16, depth: 1)

        // How many thread groups needed to cover the texture
        let threadGroups = MTLSize(
            width:  (input.width  + threadGroupSize.width  - 1) / threadGroupSize.width,
            height: (input.height + threadGroupSize.height - 1) / threadGroupSize.height,
            depth: 1
        )

        encoder.dispatchThreadgroups(threadGroups, threadsPerThreadgroup: threadGroupSize)
        encoder.endEncoding()

        commandBuffer.commit()
        commandBuffer.waitUntilCompleted()  // Synchronous — or use addCompletedHandler for async
    }
}

    Choosing Thread Group Size

    // Option 1: Fixed size (simple, works everywhere)
let threadGroupSize = MTLSize(width: 16, height: 16, depth: 1)

// Option 2: Optimal size from pipeline (Apple recommended)
let w = pipeline.threadExecutionWidth  // Usually 32 on Apple GPUs
let h = pipeline.maxTotalThreadsPerThreadgroup / w
let threadGroupSize = MTLSize(width: w, height: h, depth: 1)

// Option 3: dispatchThreads (automatic, iOS 11+)
// Let Metal figure out the optimal grouping
encoder.dispatchThreads(
    MTLSize(width: texture.width, height: texture.height, depth: 1),
    threadsPerThreadgroup: threadGroupSize
)
// With this, you STILL need the bounds check in the kernel

    Threads within a threadgroup can share data via threadgroup memory. This is much faster than reading from textures or device memory.

    texture2d<float, access::read>  inTex  <span class="xref-missing" title="note not in this site">texture(0)</span>,
    texture2d<float, access::write> outTex <span class="xref-missing" title="note not in this site">texture(1)</span>,
    uint2 gid <span class="xref-missing" title="note not in this site">thread_position_in_grid</span>,
    uint2 tid <span class="xref-missing" title="note not in this site">thread_position_in_threadgroup</span>)
{
    // Shared tile: 18x18 for a 16x16 group with 1-pixel border
    threadgroup float4 tile[18][18];

    // Each thread loads its pixel into shared memory
    int2 loadPos = int2(gid) - 1 + int2(tid);
    loadPos = clamp(loadPos, int2(0), int2(inTex.get_width()-1, inTex.get_height()-1));
    tile[tid.y + 1][tid.x + 1] = inTex.read(uint2(loadPos));

    // Border threads load extra pixels
    if (tid.x == 0)  tile[tid.y + 1][0] = inTex.read(uint2(max(int(gid.x)-1, 0), gid.y));
    if (tid.x == 15) tile[tid.y + 1][17] = inTex.read(uint2(min(gid.x+1, inTex.get_width()-1), gid.y));
    if (tid.y == 0)  tile[0][tid.x + 1] = inTex.read(uint2(gid.x, max(int(gid.y)-1, 0)));
    if (tid.y == 15) tile[17][tid.x + 1] = inTex.read(uint2(gid.x, min(gid.y+1, inTex.get_height()-1)));

    // BARRIER: wait for all threads to finish loading
    threadgroup_barrier(mem_flags::mem_threadgroup);

    // Now compute 3x3 box blur from shared memory (fast!)
    float4 sum = float4(0);
    for (int y = 0; y < 3; y++) {
        for (int x = 0; x < 3; x++) {
            sum += tile[tid.y + y][tid.x + x];
        }
    }
    sum /= 9.0;

    outTex.write(sum, gid);
}

    threadgroup_barrier

    This is critical: it ensures ALL threads in the group have reached this point before any thread continues. Without it, you might read data that another thread hasn't written yet.

    // All threads load data
tile[tid.y][tid.x] = someData;

// Wait for everyone
threadgroup_barrier(mem_flags::mem_threadgroup);

// Now safe to read any position in tile
float4 neighborData = tile[tid.y + 1][tid.x];

    Struct Alignment Rules

    Metal and Swift must agree on memory layout. Key rules:

    float    = 4 bytes, align to 4
float2   = 8 bytes, align to 8
float3   = 16 bytes, align to 16 (NOT 12!)
float4   = 16 bytes, align to 16
float4x4 = 64 bytes, align to 16
bool     = avoid (use int or float instead)

    var color: SIMD4<Float>     // 16 bytes

    Method 3: Triple Buffering (Smooth Animation)

    For data that updates every frame, use 3 buffers to avoid GPU/CPU conflicts:

    class TripleBufferedRenderer {
    let buffers: [MTLBuffer]
    var currentBuffer = 0
    let semaphore = DispatchSemaphore(value: 3)  // Allow 3 frames in flight

    init(device: MTLDevice, size: Int) {
        buffers = (0..<3).map { _ in
            device.makeBuffer(length: size, options: .storageModeShared)!
        }
    }

    func updateAndDraw(encoder: MTLRenderCommandEncoder, params: Params) {
        semaphore.wait()  // Wait if 3 frames are already in flight

        // Write to current buffer (GPU is reading from the other 2)
        let buffer = buffers[currentBuffer]
        buffer.contents().storeBytes(of: params, as: Params.self)

        encoder.setFragmentBuffer(buffer, offset: 0, index: 0)

        // After GPU finishes, signal the semaphore
        let sem = semaphore
        encoder.parentCommandBuffer?.addCompletedHandler { _ in
            sem.signal()
        }

        currentBuffer = (currentBuffer + 1) % 3
    }
}

    13 — Blur Effects

    Blur is the most important visual effect in iOS — used in navigation bars, sheets, Control Center, notifications, and every glass material. This lesson builds from the simplest blur to Apple-quality implementations.

    Why Blur is Hard

    A naive blur reads many neighboring pixels for every output pixel. For a 20px radius, that's (2*20+1)^2 = 1681 texture reads per pixel. At 2.7 million pixels, that's 4.5 billion reads per frame. We need smart algorithms to make this fast.

    Level 1: Box Blur (Simplest)

    Average all pixels in a square region. Simple but produces boxy artifacts.

    fragment half4 boxBlur(
    VertexOut in <span class="xref-missing" title="note not in this site">stage_in</span>,
    texture2d<half> tex <span class="xref-missing" title="note not in this site">texture(0)</span>,
    constant float &radius <span class="xref-missing" title="note not in this site">buffer(0)</span>,
    constant float2 &texSize <span class="xref-missing" title="note not in this site">buffer(1)</span>)
{
    constexpr sampler s(filter::linear, address::clamp_to_edge);

    half4 sum = half4(0);
    int r = int(radius);
    float count = 0;

    for (int y = -r; y <= r; y++) {
        for (int x = -r; x <= r; x++) {
            float2 offset = float2(x, y) / texSize;
            sum += tex.sample(s, in.uv + offset);
            count += 1.0;
        }
    }

    return sum / half(count);
}

    Cost: O(radius^2) per pixel. Unusable for large radii.

    Level 2: Separable Blur (2-Pass)

    Key insight: a 2D Gaussian can be decomposed into two 1D passes (horizontal then vertical). This reduces O(r^2) to O(2r).

    // Pass 1: Horizontal blur
fragment half4 blurH(
    VertexOut in <span class="xref-missing" title="note not in this site">stage_in</span>,
    texture2d<half> tex <span class="xref-missing" title="note not in this site">texture(0)</span>,
    constant float &radius <span class="xref-missing" title="note not in this site">buffer(0)</span>,
    constant float2 &texSize <span class="xref-missing" title="note not in this site">buffer(1)</span>)
{
    constexpr sampler s(filter::linear, address::clamp_to_edge);

    half4 sum = half4(0);
    float totalWeight = 0;
    int r = int(radius);
    float sigma = radius * 0.5;

    for (int x = -r; x <= r; x++) {
        float weight = exp(-float(x * x) / (2.0 * sigma * sigma));
        float2 offset = float2(float(x) / texSize.x, 0.0);
        sum += tex.sample(s, in.uv + offset) * half(weight);
        totalWeight += weight;
    }

    return sum / half(totalWeight);
}

// Pass 2: Vertical blur (same shader, different direction)
fragment half4 blurV(
    VertexOut in <span class="xref-missing" title="note not in this site">stage_in</span>,
    texture2d<half> tex <span class="xref-missing" title="note not in this site">texture(0)</span>,
    constant float &radius <span class="xref-missing" title="note not in this site">buffer(0)</span>,
    constant float2 &texSize <span class="xref-missing" title="note not in this site">buffer(1)</span>)
{
    constexpr sampler s(filter::linear, address::clamp_to_edge);

    half4 sum = half4(0);
    float totalWeight = 0;
    int r = int(radius);
    float sigma = radius * 0.5;

    for (int y = -r; y <= r; y++) {
        float weight = exp(-float(y * y) / (2.0 * sigma * sigma));
        float2 offset = float2(0.0, float(y) / texSize.y);
        sum += tex.sample(s, in.uv + offset) * half(weight);
        totalWeight += weight;
    }

    return sum / half(totalWeight);
}

encoder1.setFragmentTexture(sourceTexture, index: 0)
// Render to intermediateTexture

    Level 3: Optimized Gaussian (Linear Sampling Trick)

    By sampling between two pixels (filter::linear), you get the interpolated value of both — effectively 2 samples for the price of 1. This halves the number of texture reads.

    // Precomputed for 9-tap kernel (radius ~4)
// These offsets sample BETWEEN pixels to get two taps in one read
constant float offsets[3] = { 0.0, 1.3846153846, 3.2307692308 };
constant float weights[3] = { 0.2270270270, 0.3162162162, 0.0702702703 };

fragment half4 optimizedBlurH(
    VertexOut in <span class="xref-missing" title="note not in this site">stage_in</span>,
    texture2d<half> tex <span class="xref-missing" title="note not in this site">texture(0)</span>,
    constant float2 &texSize <span class="xref-missing" title="note not in this site">buffer(0)</span>)
{
    constexpr sampler s(filter::linear, address::clamp_to_edge);
    float texelSize = 1.0 / texSize.x;

    half4 color = tex.sample(s, in.uv) * half(weights[0]);

    for (int i = 1; i < 3; i++) {
        float offset = offsets[i] * texelSize;
        color += tex.sample(s, in.uv + float2(offset, 0)) * half(weights[i]);
        color += tex.sample(s, in.uv - float2(offset, 0)) * half(weights[i]);
    }

    return color;
}

    5 texture reads instead of 9 for the same quality. At larger radii, the savings compound.

    Level 4: Kawase Blur (Best Quality/Performance Ratio)

    Used by many game engines and real-time renderers. Instead of one large-radius blur, apply multiple small blurs at increasing sample distances. Each pass is just 5 taps.

    fragment half4 kawaseBlur(
    VertexOut in <span class="xref-missing" title="note not in this site">stage_in</span>,
    texture2d<half> tex <span class="xref-missing" title="note not in this site">texture(0)</span>,
    constant float &iteration <span class="xref-missing" title="note not in this site">buffer(0)</span>,  // 0, 1, 2, 3...
    constant float2 &texSize <span class="xref-missing" title="note not in this site">buffer(1)</span>)
{
    constexpr sampler s(filter::linear, address::clamp_to_edge);

    float2 texelSize = 1.0 / texSize;
    float offset = float(iteration) + 0.5;

    half4 color = tex.sample(s, in.uv) * 4.0h;
    color += tex.sample(s, in.uv + float2(-offset, -offset) * texelSize);
    color += tex.sample(s, in.uv + float2( offset, -offset) * texelSize);
    color += tex.sample(s, in.uv + float2(-offset,  offset) * texelSize);
    color += tex.sample(s, in.uv + float2( offset,  offset) * texelSize);

    return color / 8.0h;
}

    Swift: Run 4-6 passes, ping-ponging between two textures:

    for i in 0..<6 {
    var iteration = Float(i)
    encoder.setFragmentBytes(&iteration, length: 4, index: 0)
    encoder.setFragmentTexture(i % 2 == 0 ? texA : texB, index: 0)
    // Render to the OTHER texture
}

    5 texture reads per pass x 6 passes = 30 reads for a large blur. Compare to 1681 reads for naive box blur.

    Level 5: Dual Kawase Blur (Apple-Quality)

    The technique used by many production renderers. Downscale with blur, then upscale with blur. Very fast, very smooth.

    // Downsample pass (halves resolution + blurs)
fragment half4 dualKawaseDown(
    VertexOut in <span class="xref-missing" title="note not in this site">stage_in</span>,
    texture2d<half> tex <span class="xref-missing" title="note not in this site">texture(0)</span>,
    constant float2 &texelSize <span class="xref-missing" title="note not in this site">buffer(0)</span>)
{
    constexpr sampler s(filter::linear, address::clamp_to_edge);
    float2 halfTexel = texelSize * 0.5;

    half4 sum = tex.sample(s, in.uv) * 4.0h;
    sum += tex.sample(s, in.uv - halfTexel);
    sum += tex.sample(s, in.uv + halfTexel);
    sum += tex.sample(s, in.uv + float2(halfTexel.x, -halfTexel.y));
    sum += tex.sample(s, in.uv - float2(halfTexel.x, -halfTexel.y));

    return sum / 8.0h;
}

// Upsample pass (doubles resolution + blurs)
fragment half4 dualKawaseUp(
    VertexOut in <span class="xref-missing" title="note not in this site">stage_in</span>,
    texture2d<half> tex <span class="xref-missing" title="note not in this site">texture(0)</span>,
    constant float2 &texelSize <span class="xref-missing" title="note not in this site">buffer(0)</span>)
{
    constexpr sampler s(filter::linear, address::clamp_to_edge);

    half4 sum = half4(0);
    sum += tex.sample(s, in.uv + float2(-texelSize.x * 2.0, 0)) ;
    sum += tex.sample(s, in.uv + float2(-texelSize.x, texelSize.y)) * 2.0h;
    sum += tex.sample(s, in.uv + float2(0, texelSize.y * 2.0));
    sum += tex.sample(s, in.uv + float2(texelSize.x, texelSize.y)) * 2.0h;
    sum += tex.sample(s, in.uv + float2(texelSize.x * 2.0, 0));
    sum += tex.sample(s, in.uv + float2(texelSize.x, -texelSize.y)) * 2.0h;
    sum += tex.sample(s, in.uv + float2(0, -texelSize.y * 2.0));
    sum += tex.sample(s, in.uv + float2(-texelSize.x, -texelSize.y)) * 2.0h;

    return sum / 12.0h;
}

    Pipeline: Source -> Down1 (1/2) -> Down2 (1/4) -> Down3 (1/8) -> Up3 (1/4) -> Up2 (1/2) -> Up1 (full)

    14 — Distortion Effects

    Distortion shaders move pixels around without changing their color. They remap where each pixel reads its source data from.

    The Concept

    Normal:     Read pixel at position (x, y)
Distorted:  Read pixel at position (x + offset_x, y + offset_y)

    The fragment shader calculates the offset and samples the original texture at the shifted position.

    Ripple

    <span class="xref-missing" title="note not in this site">stitchable</span> float2 ripple(float2 position, float2 origin, float time) {
    float dist = distance(position, origin);
    float wave = sin(dist * 0.1 - time * 8.0) * 8.0;
    float falloff = exp(-dist * 0.01) * exp(-time * 1.5);
    float2 dir = normalize(position - origin + 0.001);
    return position + dir * wave * falloff;
}

    Wave

    <span class="xref-missing" title="note not in this site">stitchable</span> float2 wave(float2 position, float time, float amplitude, float frequency) {
    float offsetY = sin(position.x * frequency * 0.01 + time * 3.0) * amplitude;
    float offsetX = cos(position.y * frequency * 0.01 + time * 2.0) * amplitude * 0.5;
    return position + float2(offsetX, offsetY);
}

    Swirl

    <span class="xref-missing" title="note not in this site">stitchable</span> float2 swirl(float2 position, float2 center, float radius, float angle) {
    float2 diff = position - center;
    float dist = length(diff);

    if (dist > radius) return position;

    float percent = (radius - dist) / radius;
    float theta = percent * percent * angle;

    float s = sin(theta);
    float c = cos(theta);
    float2 rotated = float2(
        diff.x * c - diff.y * s,
        diff.x * s + diff.y * c
    );

    return center + rotated;
}

    Barrel Distortion (Lens Effect)

    <span class="xref-missing" title="note not in this site">stitchable</span> float2 barrel(float2 position, float2 center, float2 size, float strength) {
    float2 uv = (position - center) / (size * 0.5);
    float r2 = dot(uv, uv);
    float distortion = 1.0 + r2 * strength;
    return center + uv * distortion * (size * 0.5);
}

    Pixelation

    <span class="xref-missing" title="note not in this site">stitchable</span> float2 pixelate(float2 position, float blockSize) {
    return floor(position / blockSize) * blockSize + blockSize * 0.5;
}

    Magnifying Glass

    <span class="xref-missing" title="note not in this site">stitchable</span> float2 magnify(float2 position, float2 center, float radius, float zoom) {
    float dist = distance(position, center);
    if (dist > radius) return position;

    float2 diff = position - center;
    float smoothEdge = smoothstep(radius * 0.8, radius, dist);
    float2 zoomed = center + diff / mix(zoom, 1.0, smoothEdge);
    return zoomed;
}

    Shockwave

    <span class="xref-missing" title="note not in this site">stitchable</span> float2 shockwave(float2 position, float2 center, float time,
                                  float waveWidth, float amplitude) {
    float dist = distance(position, center);
    float radius = time * 300.0;  // Expanding ring

    float diff = abs(dist - radius);
    if (diff > waveWidth) return position;

    float intensity = (1.0 - diff / waveWidth) * amplitude;
    intensity *= 1.0 - smoothstep(0.0, 1.0, time);  // Fade over time

    float2 dir = normalize(position - center);
    return position + dir * sin(diff * 0.5) * intensity;
}

    15 — Noise and Procedural Patterns

    Noise functions generate organic, natural-looking randomness. They're the foundation of clouds, fire, water, terrain, and countless visual effects.

    Random (Hash) Function

    Pure randomness — different value for every position, no smoothness:

    float random(float2 st) {
    return fract(sin(dot(st, float2(12.9898, 78.233))) * 43758.5453123);
}

// Returns 0.0 to 1.0
// Same input always gives same output (deterministic)
// Adjacent pixels get completely different values (not smooth)

    Value Noise

    Smooth random values — interpolate between random grid points:

    float valueNoise(float2 st) {
    float2 i = floor(st);
    float2 f = fract(st);

    // Smooth interpolation curve
    float2 u = f * f * (3.0 - 2.0 * f);  // smoothstep

    // Random values at 4 grid corners
    float a = random(i);
    float b = random(i + float2(1.0, 0.0));
    float c = random(i + float2(0.0, 1.0));
    float d = random(i + float2(1.0, 1.0));

    // Bilinear interpolation
    return mix(mix(a, b, u.x), mix(c, d, u.x), u.y);
}

    Perlin-Style Gradient Noise

    Smoother than value noise, more organic:

    float2 gradientDir(float2 p) {
    p = fmod(p, 289.0);
    float x = fract(sin(dot(p, float2(127.1, 311.7))) * 43758.5453);
    float y = fract(sin(dot(p, float2(269.5, 183.3))) * 43758.5453);
    return normalize(float2(x, y) * 2.0 - 1.0);
}

float gradientNoise(float2 st) {
    float2 i = floor(st);
    float2 f = fract(st);
    float2 u = f * f * f * (f * (f * 6.0 - 15.0) + 10.0);  // Improved smoothstep

    float2 ga = gradientDir(i);
    float2 gb = gradientDir(i + float2(1.0, 0.0));
    float2 gc = gradientDir(i + float2(0.0, 1.0));
    float2 gd = gradientDir(i + float2(1.0, 1.0));

    float va = dot(ga, f);
    float vb = dot(gb, f - float2(1.0, 0.0));
    float vc = dot(gc, f - float2(0.0, 1.0));
    float vd = dot(gd, f - float2(1.0, 1.0));

    return mix(mix(va, vb, u.x), mix(vc, vd, u.x), u.y) * 0.5 + 0.5;
}

    Fractal Brownian Motion (FBM)

    Layer multiple octaves of noise for detail at multiple scales:

    float fbm(float2 st, int octaves) {
    float value = 0.0;
    float amplitude = 0.5;
    float frequency = 1.0;

    for (int i = 0; i < octaves; i++) {
        value += amplitude * gradientNoise(st * frequency);
        frequency *= 2.0;     // Each octave is 2x frequency
        amplitude *= 0.5;     // Each octave is half amplitude
    }

    return value;
}

// Usage: fbm(uv * 4.0, 6) -> cloud-like pattern

    Voronoi (Cellular Noise)

    Creates cell-like patterns (stone tiles, scales, bubbles):

    float voronoi(float2 st) {
    float2 i = floor(st);
    float2 f = fract(st);

    float minDist = 1.0;

    // Check 3x3 neighborhood
    for (int y = -1; y <= 1; y++) {
        for (int x = -1; x <= 1; x++) {
            float2 neighbor = float2(x, y);
            float2 point = random2(i + neighbor);  // Random point in each cell
            float dist = distance(f, neighbor + point);
            minDist = min(minDist, dist);
        }
    }

    return minDist;
}

float2 random2(float2 st) {
    return float2(
        fract(sin(dot(st, float2(127.1, 311.7))) * 43758.5453),
        fract(sin(dot(st, float2(269.5, 183.3))) * 43758.5453)
    );
}

    Practical Uses

    Cloud/Fog

    fragment half4 clouds(VertexOut in <span class="xref-missing" title="note not in this site">stage_in</span>, constant float &time <span class="xref-missing" title="note not in this site">buffer(0)</span>) {
    float2 uv = in.uv * 4.0;
    uv += float2(time * 0.1, time * 0.05);  // Drift

    float n = fbm(uv, 6);
    half3 skyColor = half3(0.4, 0.6, 0.9);
    half3 cloudColor = half3(1.0);

    return half4(mix(skyColor, cloudColor, half(n)), 1.0h);
}

    Dissolve Transition

    <span class="xref-missing" title="note not in this site">stitchable</span> half4 dissolve(float2 pos, half4 color, float progress) {
    float noise = random(pos * 0.05);
    float threshold = progress * 1.2;  // Slightly over 1.0 for full dissolve

    if (noise < threshold) {
        // Glowing edge
        if (noise > threshold - 0.05) {
            return half4(1.0, 0.5, 0.0, color.a);  // Orange edge
        }
        return half4(0, 0, 0, 0);  // Dissolved
    }
    return color;
}

    16 — Blending and Compositing

    How to combine layers — the math behind Photoshop blend modes, alpha compositing, and Apple's private compositingFilter.

    Alpha Compositing (Porter-Duff)

    The foundation. How a source (foreground) combines with a destination (background):

    // Source Over (Normal blending — most common)
float4 sourceOver(float4 src, float4 dst) {
    return src + dst * (1.0 - src.a);
}

// Premultiplied version (what Metal actually uses):
// result = src + dst * (1 - src.a)
// This works directly because src.rgb is already multiplied by src.a

    All 12 Porter-Duff Modes

    float4 pdClear(float4 s, float4 d)      { return float4(0); }
float4 pdCopy(float4 s, float4 d)       { return s; }
float4 pdSourceOver(float4 s, float4 d)  { return s + d * (1-s.a); }
float4 pdSourceIn(float4 s, float4 d)    { return s * d.a; }
float4 pdSourceOut(float4 s, float4 d)   { return s * (1-d.a); }
float4 pdSourceAtop(float4 s, float4 d)  { return s * d.a + d * (1-s.a); }
float4 pdDestOver(float4 s, float4 d)    { return d + s * (1-d.a); }
float4 pdDestIn(float4 s, float4 d)      { return d * s.a; }
float4 pdDestOut(float4 s, float4 d)     { return d * (1-s.a); }
float4 pdDestAtop(float4 s, float4 d)    { return d * s.a + s * (1-s.a); }
float4 pdXor(float4 s, float4 d)         { return s*(1-d.a) + d*(1-s.a); }
float4 pdPlusLighter(float4 s, float4 d) { return min(s + d, 1.0); }

    Apple's plusL compositing filter = pdPlusLighter. This is their go-to for glass highlights.

    Photoshop-Style Blend Modes

    These operate on the RGB channels (alpha handled separately):

    // Multiply (darkens)
float3 multiply(float3 base, float3 blend) { return base * blend; }

// Screen (lightens)
float3 screen(float3 base, float3 blend) { return 1.0 - (1.0-base) * (1.0-blend); }

// Overlay (contrast: darkens darks, lightens lights)
float3 overlay(float3 base, float3 blend) {
    return mix(
        2.0 * base * blend,
        1.0 - 2.0 * (1.0-base) * (1.0-blend),
        step(0.5, base)
    );
}

// Soft Light (subtle overlay)
float3 softLight(float3 base, float3 blend) {
    return mix(
        2.0 * base * blend + base * base * (1.0 - 2.0*blend),
        sqrt(base) * (2.0*blend - 1.0) + 2.0*base * (1.0-blend),
        step(0.5, blend)
    );
}

// Hard Light
float3 hardLight(float3 base, float3 blend) { return overlay(blend, base); }

// Color Dodge (brightens base by blend)
float3 colorDodge(float3 base, float3 blend) {
    return base / max(1.0 - blend, 0.001);
}

// Color Burn (darkens base by blend)
float3 colorBurn(float3 base, float3 blend) {
    return 1.0 - (1.0-base) / max(blend, 0.001);
}

// Difference
float3 difference(float3 base, float3 blend) { return abs(base - blend); }

// Exclusion
float3 exclusion(float3 base, float3 blend) {
    return base + blend - 2.0*base*blend;
}

// Add (Linear Dodge)
float3 add(float3 base, float3 blend) { return min(base + blend, 1.0); }

    Using Blend Modes with Alpha

    float4 blendWithAlpha(float4 base, float4 blend, float3 (*blendFunc)(float3, float3)) {
    float3 blended = blendFunc(base.rgb, blend.rgb);
    float3 result = mix(base.rgb, blended, blend.a);
    return float4(result, base.a);
}

    On CALayer, applied via layer.setValue("plusL", forKey: "compositingFilter"):

class Renderer: NSObject, MTKViewDelegate {
    let device: MTLDevice
    let commandQueue: MTLCommandQueue
    let pipelineState: MTLRenderPipelineState
    var time: Float = 0

    init(metalView: MTKView) {
        device = metalView.device!
        commandQueue = device.makeCommandQueue()!

        let library = device.makeDefaultLibrary()!
        let descriptor = MTLRenderPipelineDescriptor()
        descriptor.vertexFunction = library.makeFunction(name: "fullscreenVertex")
        descriptor.fragmentFunction = library.makeFunction(name: "myEffect")
        descriptor.colorAttachments[0].pixelFormat = metalView.colorPixelFormat

        pipelineState = try! device.makeRenderPipelineState(descriptor: descriptor)

        super.init()
        metalView.delegate = self
    }

    func draw(in view: MTKView) {
        time += 1.0 / Float(view.preferredFramesPerSecond)

        guard let drawable = view.currentDrawable,
              let descriptor = view.currentRenderPassDescriptor else { return }

        let buffer = commandQueue.makeCommandBuffer()!
        let encoder = buffer.makeRenderCommandEncoder(descriptor: descriptor)!

        encoder.setRenderPipelineState(pipelineState)
        encoder.setFragmentBytes(&time, length: 4, index: 0)
        encoder.drawPrimitives(type: .triangle, vertexStart: 0, vertexCount: 3)

        encoder.endEncoding()
        buffer.present(drawable)
        buffer.commit()
    }

    func mtkView(_ view: MTKView, drawableSizeWillChange size: CGSize) {}
}

    let desc = MTLTextureDescriptor.texture2DDescriptor(

    enc1.setFragmentBytes(&dir, length: 8, index: 0)
    enc1.drawPrimitives(type: .triangle, vertexStart: 0, vertexCount: 3)
    enc1.endEncoding()

    // Pass 2: Vertical blur (intermediate -> output)
    let pass2Desc = MTLRenderPassDescriptor()
    pass2Desc.colorAttachments[0].texture = output
    pass2Desc.colorAttachments[0].loadAction = .dontCare
    pass2Desc.colorAttachments[0].storeAction = .store

    let enc2 = buffer.makeRenderCommandEncoder(descriptor: pass2Desc)!
    enc2.setRenderPipelineState(blurVPipeline)
    enc2.setFragmentTexture(intermediate, index: 0)

}

    Distortion Effects

    Receives a position, returns a new position to sample from. Shifts where each pixel reads its color from. Cannot change colors, only positions.

    Metal Shader

    // Returns a float2 — the new sample position
<span class="xref-missing" title="note not in this site">stitchable</span> float2 ripple(float2 position, float2 origin, float time,
                              float amplitude, float frequency, float decay) {
    float dist = distance(position, origin);
    float wave = sin(dist * frequency - time * 10.0);
    float falloff = exp(-dist * decay) * exp(-time * 2.0);
    float2 direction = normalize(position - origin + 0.001);

    return position + direction * wave * amplitude * falloff;
}

<span class="xref-missing" title="note not in this site">stitchable</span> float2 pixelate(float2 position, float size) {
    return floor(position / size) * size + size * 0.5;
}

<span class="xref-missing" title="note not in this site">stitchable</span> float2 fisheye(float2 position, float2 center, float radius, float strength) {
    float2 diff = position - center;
    float dist = length(diff);

    if (dist < radius) {
        float percent = dist / radius;
        float distortion = pow(percent, strength);
        return center + diff * distortion / percent;
    }
    return position;
}

<span class="xref-missing" title="note not in this site">stitchable</span> half4 progressiveBlur(float2 position, SwiftUI::Layer layer,
                                      float2 size, float maxRadius) {
    // Blur increases from top to bottom
    float progress = position.y / size.y;
    float radius = progress * maxRadius;

    if (radius < 1.0) return layer.sample(position);

    half4 sum = half4(0);
    float totalWeight = 0;
    int r = int(min(radius, 20.0));  // Cap for performance

    for (int y = -r; y <= r; y++) {
        for (int x = -r; x <= r; x++) {
            float weight = exp(-float(x*x + y*y) / (2.0 * radius * radius));
            sum += layer.sample(position + float2(x, y)) * half(weight);
            totalWeight += weight;
        }
    }

    return sum / half(totalWeight);
}

    20 — Variable Blur

    Different blur radius at different positions — the technique behind Apple's scroll edge blur, depth-of-field, and tilt-shift effects.

    Concept

    Standard blur: every pixel gets the same blur radius. Variable blur: each pixel's blur radius is controlled by a mask or formula.

    Mask image:       Result:
█████░░░░░        [blurry][sharp]
█████░░░░░   ->   [blurry][sharp]
█████░░░░░        [blurry][sharp]

White = full blur, Black = no blur

    Method 1: Mask-Driven (Texture Mask)

    <span class="xref-missing" title="note not in this site">stitchable</span> half4 variableBlur(
    float2 position, SwiftUI::Layer layer,
    float2 size, float maxRadius, float maskY)
{
    // Linear mask: blur increases toward bottom
    float maskValue = smoothstep(maskY, size.y, position.y);
    float radius = maskValue * maxRadius;

    if (radius < 0.5) return layer.sample(position);

    half4 sum = half4(0);
    float totalWeight = 0;
    int r = int(min(radius, 16.0));

    for (int y = -r; y <= r; y++) {
        for (int x = -r; x <= r; x++) {
            float dist = length(float2(x, y));
            if (dist > radius) continue;

            float weight = 1.0 - (dist / radius);  // Linear falloff
            sum += layer.sample(position + float2(x, y)) * half(weight);
            totalWeight += weight;
        }
    }

    return sum / half(totalWeight);
}

    Method 2: Radial Variable Blur

    // Tilt-shift / miniature effect
<span class="xref-missing" title="note not in this site">stitchable</span> half4 tiltShift(
    float2 position, SwiftUI::Layer layer,
    float2 size, float focusY, float focusRange, float maxBlur)
{
    float dist = abs(position.y - focusY);
    float blur = smoothstep(focusRange * 0.5, focusRange, dist) * maxBlur;

    if (blur < 0.5) return layer.sample(position);

    half4 sum = half4(0);
    float count = 0;
    int r = int(min(blur, 12.0));

    for (int y = -r; y <= r; y++) {
        for (int x = -r; x <= r; x++) {
            if (length(float2(x,y)) > blur) continue;
            sum += layer.sample(position + float2(x, y));
            count += 1.0;
        }
    }

    return sum / half(count);
}

    21 — Glass and Refraction

    The techniques behind Apple's frosted glass materials, Liquid Glass, and light refraction effects.

    Frosted Glass Recipe

    Apple's glass = backdrop blur + saturation boost + tint + specular highlight + compositing.

    <span class="xref-missing" title="note not in this site">stitchable</span> half4 frostedGlass(
    float2 position, SwiftUI::Layer layer,
    float2 size, float blurRadius, half4 tintColor)
{
    // 1. Blur the background
    half4 blurred = half4(0);
    float totalWeight = 0;
    int r = int(min(blurRadius, 12.0));

    for (int y = -r; y <= r; y++) {
        for (int x = -r; x <= r; x++) {
            float weight = exp(-float(x*x + y*y) / (2.0 * blurRadius * blurRadius));
            blurred += layer.sample(position + float2(x, y)) * half(weight);
            totalWeight += weight;
        }
    }
    blurred /= half(totalWeight);

    // 2. Boost saturation (Apple uses ~1.8x)
    half luma = dot(blurred.rgb, half3(0.2126, 0.7152, 0.0722));
    blurred.rgb = mix(half3(luma), blurred.rgb, 1.8h);

    // 3. Apply tint
    blurred.rgb = mix(blurred.rgb, tintColor.rgb, tintColor.a);

    // 4. Add subtle highlight (plusL compositing)
    blurred.rgb = min(blurred.rgb + 0.08h, 1.0h);

    return half4(blurred.rgb, 0.92h);
}

    Refraction (Light Bending)

    <span class="xref-missing" title="note not in this site">stitchable</span> half4 refraction(
    float2 position, SwiftUI::Layer layer,
    float2 size, float2 center, float radius, float ior)
{
    float2 diff = position - center;
    float dist = length(diff);

    if (dist > radius) return layer.sample(position);

    // Surface normal of the glass sphere
    float2 normal = diff / radius;
    float nz = sqrt(max(1.0 - dot(normal, normal), 0.0));

    // Snell's law refraction
    float eta = 1.0 / ior;  // ior: index of refraction (glass ~1.5)
    float3 incident = float3(0, 0, -1);
    float3 n = float3(normal, nz);
    float cosI = dot(-incident, n);
    float sinT2 = eta * eta * (1.0 - cosI * cosI);
    float3 refracted = eta * incident + (eta * cosI - sqrt(max(1.0 - sinT2, 0.0))) * n;

    // Offset the sample position
    float2 offset = refracted.xy * radius * 0.3;
    return layer.sample(position + offset);
}

    Chromatic Aberration

    Light splits into colors at glass edges:

    <span class="xref-missing" title="note not in this site">stitchable</span> half4 chromaticAberration(
    float2 position, SwiftUI::Layer layer,
    float2 size, float strength)
{
    float2 center = size * 0.5;
    float2 dir = (position - center) / size;
    float dist = length(dir);

    float2 offset = normalize(dir) * dist * strength;

    half r = layer.sample(position + offset).r;
    half g = layer.sample(position).g;
    half b = layer.sample(position - offset).b;
    half a = layer.sample(position).a;

    return half4(r, g, b, a);
}

    Liquid Glass (iOS 26-Style)

    Combines refraction + blur + specular + Fresnel:

    <span class="xref-missing" title="note not in this site">stitchable</span> half4 liquidGlass(
    float2 position, SwiftUI::Layer layer,
    float2 size, float2 glassCenter, float2 glassSize, float cornerRadius, float time)
{
    // Rounded rect SDF
    float2 p = abs(position - glassCenter) - glassSize * 0.5 + cornerRadius;
    float d = length(max(p, 0.0)) + min(max(p.x, p.y), 0.0) - cornerRadius;

    if (d > 2.0) return layer.sample(position);

    // Glass surface normal (from SDF gradient)
    float eps = 1.0;
    float dx = length(max(abs(position + float2(eps,0) - glassCenter) - glassSize*0.5 + cornerRadius, 0.0))
             - length(max(abs(position - float2(eps,0) - glassCenter) - glassSize*0.5 + cornerRadius, 0.0));
    float dy = length(max(abs(position + float2(0,eps) - glassCenter) - glassSize*0.5 + cornerRadius, 0.0))
             - length(max(abs(position - float2(0,eps) - glassCenter) - glassSize*0.5 + cornerRadius, 0.0));
    float2 normal = normalize(float2(dx, dy));

    // Refraction offset
    float refractionAmount = smoothstep(2.0, -10.0, d) * 12.0;
    float2 refracted = position + normal * refractionAmount;

    // Blur the refracted sample
    half4 blurred = half4(0);
    for (int y = -4; y <= 4; y++) {
        for (int x = -4; x <= 4; x++) {
            blurred += layer.sample(refracted + float2(x, y) * 1.5);
        }
    }
    blurred /= 81.0h;

    // Saturation boost
    half luma = dot(blurred.rgb, half3(0.2126, 0.7152, 0.0722));
    blurred.rgb = mix(half3(luma), blurred.rgb, 1.6h);

    // Specular highlight (animated light direction)
    float2 lightDir = normalize(float2(cos(time * 0.5), sin(time * 0.3)));
    float specular = pow(max(dot(normal, lightDir), 0.0), 24.0);

    // Fresnel (edges darker/more reflective)
    float fresnel = smoothstep(-8.0, 0.0, d);

    // Compose
    blurred.rgb += half(specular * 0.25);
    blurred.rgb *= half(1.0 - fresnel * 0.25);

    // Edge glow
    float edgeGlow = smoothstep(2.0, 0.0, abs(d)) * 0.15;
    blurred.rgb += half(edgeGlow);

    // Smooth edge alpha
    float alpha = smoothstep(2.0, -1.0, d);
    blurred.a = half(alpha) * 0.9h;

    return blurred;
}

    22 — Lighting and Normal Maps

    Adding depth and dimension to flat surfaces using lighting calculations and normal maps.

    Phong Lighting Model

    Three components: ambient + diffuse + specular.

    struct Light {
    float3 position;
    float3 color;
    float intensity;
};

float3 phongLighting(float3 normal, float3 viewDir, float3 lightDir,
                      float3 lightColor, float3 surfaceColor) {
    // Ambient: base illumination
    float3 ambient = surfaceColor * 0.15;

    // Diffuse: how much the surface faces the light
    float diff = max(dot(normal, lightDir), 0.0);
    float3 diffuse = surfaceColor * lightColor * diff;

    // Specular: shiny highlight
    float3 reflectDir = reflect(-lightDir, normal);
    float spec = pow(max(dot(viewDir, reflectDir), 0.0), 32.0);
    float3 specular = lightColor * spec * 0.5;

    return ambient + diffuse + specular;
}

    Normal Maps

    Normal maps store surface detail as RGB colors where R=X, G=Y, B=Z direction:

    fragment half4 normalMapped(
    VertexOut in <span class="xref-missing" title="note not in this site">stage_in</span>,
    texture2d<half> colorTex <span class="xref-missing" title="note not in this site">texture(0)</span>,
    texture2d<half> normalTex <span class="xref-missing" title="note not in this site">texture(1)</span>,
    constant Light &light <span class="xref-missing" title="note not in this site">buffer(0)</span>)
{
    constexpr sampler s(filter::linear);

    half4 color = colorTex.sample(s, in.texCoord);

    // Read normal from map (stored as 0-1, remap to -1 to 1)
    half3 normal = normalTex.sample(s, in.texCoord).rgb * 2.0h - 1.0h;
    normal = normalize(normal);

    float3 lightDir = normalize(light.position - in.worldPosition);
    float3 viewDir = normalize(float3(0, 0, 1) - in.worldPosition);

    float3 lit = phongLighting(float3(normal), viewDir, lightDir,
                                light.color, float3(color.rgb));

    return half4(half3(lit), color.a);
}

    Generating Normals from Height Map

    Create a normal map from a grayscale height texture:

    kernel void heightToNormal(
    texture2d<float, access::read> heightMap <span class="xref-missing" title="note not in this site">texture(0)</span>,
    texture2d<float, access::write> normalMap <span class="xref-missing" title="note not in this site">texture(1)</span>,
    uint2 gid <span class="xref-missing" title="note not in this site">thread_position_in_grid</span>)
{
    float l = heightMap.read(uint2(max(int(gid.x)-1, 0), gid.y)).r;
    float r = heightMap.read(uint2(min(gid.x+1, heightMap.get_width()-1), gid.y)).r;
    float u = heightMap.read(uint2(gid.x, max(int(gid.y)-1, 0))).r;
    float d = heightMap.read(uint2(gid.x, min(gid.y+1, heightMap.get_height()-1))).r;

    float3 normal = normalize(float3(l - r, u - d, 2.0));
    normalMap.write(float4(normal * 0.5 + 0.5, 1.0), gid);  // Pack to 0-1
}

    24 — Shader Performance

    How to write shaders that run at 120fps on every device. Profile, optimize, and avoid common pitfalls.

    The Golden Rules

    1. Use half instead of float — 2x faster on Apple GPUs for colors and visual effects
    2. Minimize texture reads — each read is expensive; use linear sampling tricks
    3. Avoid branching — GPUs execute BOTH branches; use mix/step instead
    4. Cap loop iterations — unbounded loops can timeout the GPU
    5. Reduce overdraw — don't shade pixels you won't see

    half vs float

    // SLOW
fragment float4 slowShader(VertexOut in <span class="xref-missing" title="note not in this site">stage_in</span>, texture2d<float> tex <span class="xref-missing" title="note not in this site">texture(0)</span>) {
    float4 color = tex.sample(s, in.uv);
    float luminance = dot(color.rgb, float3(0.2126, 0.7152, 0.0722));
    return float4(float3(luminance), color.a);
}

// FAST (2x on Apple GPUs)
fragment half4 fastShader(VertexOut in <span class="xref-missing" title="note not in this site">stage_in</span>, texture2d<half> tex <span class="xref-missing" title="note not in this site">texture(0)</span>) {
    half4 color = tex.sample(s, in.uv);
    half luminance = dot(color.rgb, half3(0.2126h, 0.7152h, 0.0722h));
    return half4(half3(luminance), color.a);
}

    Branching

    // SLOW — GPU executes both branches, discards one
if (uv.x > 0.5) {
    color = expensiveEffectA(uv);
} else {
    color = expensiveEffectB(uv);
}

// FAST — math replaces branching
float mask = step(0.5, uv.x);

// EXCEPTION: Early-out is OK when it skips large computation
if (distance > maxRadius) return layer.sample(position);  // Skip blur entirely
// This IS beneficial because ALL threads in the group skip together

    Texture Read Optimization

    // SLOW: Many redundant reads
for (int i = 0; i < 100; i++) {
    sum += texture.sample(s, uv + offset[i]);  // 100 reads!
}

// FASTER: Separable passes (2 x 10 instead of 100)
// Pass 1: 10 horizontal reads
// Pass 2: 10 vertical reads

// FASTEST: Linear sampling trick (sample between pixels)
// 1 bilinear read = 2 nearest reads
sum += texture.sample(linearSampler, uv + 1.5 * texelSize);  // Gets average of pixel 1 and 2

    Loop Optimization

    // DANGEROUS: Unbounded loop can timeout GPU
for (int i = 0; i < int(radius); i++) { ... }
// If radius = 1000, GPU will timeout/crash

// SAFE: Cap the iteration count
int r = min(int(radius), 16);  // Never more than 16 iterations
for (int i = -r; i <= r; i++) { ... }

// FASTER: Unroll small loops
// The compiler does this automatically for small constant counts
// But you can hint: #pragma unroll

    // SLOW: Random access pattern
color = texture.read(uint2(randomX, randomY));

// FAST: Sequential access (cache-friendly)
// Threads in a threadgroup should read adjacent pixels
color = texture.read(gid);  // Each thread reads its own position

threadgroup float4 tile[16][16];
tile[tid.y][tid.x] = texture.read(gid);

    Effect 5: Liquid Glass Refraction

    What Apple uses internally: custom Metal shaders in ChatKit's default.metallib.

    <span class="xref-missing" title="note not in this site">stitchable</span> half4 liquidGlass(
    float2 position, SwiftUI::Layer layer,
    float2 size, float time)
{
    float2 uv = position / size;
    float2 center = float2(0.5, 0.5);

    // SDF for rounded rectangle (the glass shape)
    float2 p = abs(uv - center) - float2(0.3, 0.15);
    float d = length(max(p, 0.0)) + min(max(p.x, p.y), 0.0) - 0.05;

    if (d > 0.02) return layer.sample(position);  // Outside glass

    // Refraction distortion (light bends at glass edges)
    float2 normal = normalize(float2(
        length(max(abs(uv + float2(0.001, 0) - center) - float2(0.3, 0.15), 0.0)) -
        length(max(abs(uv - float2(0.001, 0) - center) - float2(0.3, 0.15), 0.0)),
        length(max(abs(uv + float2(0, 0.001) - center) - float2(0.3, 0.15), 0.0)) -
        length(max(abs(uv - float2(0, 0.001) - center) - float2(0.3, 0.15), 0.0))
    ));

    float refractionStrength = smoothstep(0.02, -0.05, d) * 15.0;
    float2 refractedPos = position + normal * refractionStrength;

    // Blur the refracted sample
    half4 sum = half4(0);
    for (int y = -3; y <= 3; y++) {
        for (int x = -3; x <= 3; x++) {
            sum += layer.sample(refractedPos + float2(x, y) * 2.0);
        }
    }
    half4 blurred = sum / 49.0h;

    // Boost saturation (Apple's signature)
    half luma = dot(blurred.rgb, half3(0.2126, 0.7152, 0.0722));
    blurred.rgb = mix(half3(luma), blurred.rgb, 1.6h);

    // Specular highlight (light reflection on glass surface)
    float specular = pow(max(dot(normal, normalize(float2(-0.5, -0.7))), 0.0), 16.0);
    blurred.rgb += half(specular * 0.3);

    // Edge darkening (Fresnel)
    float fresnel = smoothstep(-0.05, 0.0, d);
    blurred.rgb *= half(1.0 - fresnel * 0.3);

    // Chromatic aberration at edges
    if (d > -0.02) {
        float abAmount = smoothstep(-0.02, 0.0, d) * 3.0;
        half r = layer.sample(refractedPos + normal * abAmount).r;
        half b = layer.sample(refractedPos - normal * abAmount).b;
        blurred.r = r;
        blurred.b = b;
    }

    return blurred;
}

    Core Shader Techniques

    1. Fresnel Conductor Equation (Real Metal Physics)

    This is NOT the cheap Schlick approximation. It uses the full Fresnel equations for conductors (metals), which have complex refractive indices (n + ik):

    vec3 fresnelConductor(float cosTheta, vec3 reflectivity, vec3 edgeTint) {
    vec3 r = clamp(reflectivity, vec3(0.0), vec3(0.99));
    vec3 r_sqrt = sqrt(r);
    // Derive n (refractive index) from reflectivity + edge tint
    vec3 n = mix((1.0 + r_sqrt) / (1.0 - r_sqrt), (1.0 - r) / (1.0 + r), edgeTint);
    // Derive k² (extinction coefficient) from n and reflectivity
    vec3 k2 = ((n + 1.0) * (n + 1.0) * r - (n - 1.0) * (n - 1.0)) / (1.0 - r);
    k2 = max(k2, vec3(0.0));
    vec3 k = sqrt(k2);
    // Full Fresnel: average of s-polarized and p-polarized reflectance
    vec3 rs_num = pow2(n) + pow2(k) - 2.0 * n * cosTheta + pow2(vec3(cosTheta));
    vec3 rs_den = pow2(n) + pow2(k) + 2.0 * n * cosTheta + pow2(vec3(cosTheta));
    vec3 rs = rs_num / rs_den;
    vec3 rp_num = (pow2(n) + pow2(k)) * pow2(vec3(cosTheta)) - 2.0 * n * cosTheta + vec3(1.0);
    vec3 rp_den = (pow2(n) + pow2(k)) * pow2(vec3(cosTheta)) + 2.0 * n * cosTheta + vec3(1.0);
    vec3 rp = rp_num / rp_den;
    return clamp(0.5 * (rs + rp), vec3(0.0), vec3(1.0));
}

    Why this matters: Real metals reflect different wavelengths at different angles. Gold looks yellow head-on but shifts warm at grazing angles. This equation captures that per-channel. Schlick approximation cannot.

    2. Perlin Noise Normal Mapping (Brushed Metal Grain)

    Generates a normal map procedurally from layered noise:

    float calculateMetallicDetail(vec2 uv, float cardType) {
    float fine   = noise(uv * 1600.0) * 0.35;   // fine grain
    float medium = noise(uv * 800.0)  * 0.3;     // medium grain

    // Directional grain (rotated 15°) for brushed look
    vec2 rotatedUV = vec2(
        uv.x * cos(PI * 0.15) - uv.y * sin(PI * 0.15),
        uv.x * sin(PI * 0.15) + uv.y * cos(PI * 0.15)
    );
    float directional = noise(rotatedUV * 1000.0) * 0.4;

}

vec3 calculateNormal(vec2 uv, float normalStrength, float cardType) {
    vec2 eps = vec2(1.0) / resolution;
    float c = calculateMetallicDetail(uv, cardType);
    float r = calculateMetallicDetail(uv + vec2(eps.x, 0.0), cardType);
    float t = calculateMetallicDetail(uv + vec2(0.0, eps.y), cardType);
    return normalize(vec3(
        (r - c) * normalStrength * 2.0,
        (t - c) * normalStrength * 2.0,

}

float hash(float2 p) {
    float3 p3 = fract(float3(p.xyx) * float3(443.897, 441.423, 437.195));
    p3 += dot(p3, p3.yzx + 19.19);
    return fract((p3.x + p3.y) * p3.z);
}

float noise(float2 p) {
    float2 i = floor(p);
    float2 f = fract(p);
    f = f * f * (3.0 - 2.0 * f);
    float a = hash(i);
    float b = hash(i + float2(1.0, 0.0));
    float c = hash(i + float2(0.0, 1.0));
    float d = hash(i + float2(1.0, 1.0));
    return mix(mix(a, b, f.x), mix(c, d, f.x), f.y);
}

// --- Fresnel Conductor ---
float3 fresnelConductor(float cosTheta, float3 reflectivity, float3 edgeTint) {
    float3 r = clamp(reflectivity, float3(0.0), float3(0.99));
    float3 r_sqrt = sqrt(r);
    float3 n = mix((1.0 + r_sqrt) / (1.0 - r_sqrt),
                   (1.0 - r) / (1.0 + r), edgeTint);
    float3 k2 = ((n + 1.0) * (n + 1.0) * r - (n - 1.0) * (n - 1.0)) / (1.0 - r);
    k2 = max(k2, float3(0.0));
    float3 k = sqrt(k2);

    float ct2 = cosTheta * cosTheta;
    float3 n2k2 = n * n + k2;
    float3 rs = (n2k2 - 2.0 * n * cosTheta + ct2)
              / (n2k2 + 2.0 * n * cosTheta + ct2);
    float3 rp = (n2k2 * ct2 - 2.0 * n * cosTheta + 1.0)
              / (n2k2 * ct2 + 2.0 * n * cosTheta + 1.0);
    return clamp(0.5 * (rs + rp), float3(0.0), float3(1.0));
}

fragment float4 cardFragment(
    float4 position <span class="xref-missing" title="note not in this site">position</span>,
    constant Uniforms &u <span class="xref-missing" title="note not in this site">buffer(0)</span>
) {
    float2 uv = position.xy / u.resolution;
    float3 viewDir = normalize(float3(uv * 2.0 - 1.0, 1.0));

    // Material selection (simplified — expand per cardType)
    float3 reflectivity = float3(0.92); // silver default
    float3 edgeTint = float3(0.94);
    float3 baseColor = float3(0.15);
    float roughness = 0.32;

    // Dynamic light (rotates opposite to card)
    float c = cos(-u.rotate), s = sin(-u.rotate);
    float3 lightDir = normalize(float3(-3.0 * c - 2.5 * s, -0.8, -3.0 * s + 2.5 * c));

    // Normal from procedural grain
    float3 normal = calcNormal(uv, 0.2, u.cardType, u.resolution);

    // Fresnel
    float NdotV = max(dot(normal, viewDir), 0.0);
    float3 F = fresnelConductor(NdotV, reflectivity, edgeTint);

    // Environment reflection
    float3 reflectDir = reflect(-viewDir, normal);
    float2 envUV = float2(atan2(reflectDir.z, reflectDir.x) / (2.0 * M_PI_F) + 0.5,
                          acos(reflectDir.y) / M_PI_F);
    float i1 = pow(max(1.0 - distance(envUV, float2(0.3, 0.4)) * 1.8, 0.0), 6.0);
    float i2 = pow(max(1.0 - distance(envUV, float2(0.7, 0.6)) * 1.8, 0.0), 6.0);
    float3 envColor = baseColor + float3(0.9) * i1 * 1.6 + float3(0.85) * i2 * 1.6;

    // Specular
    float3 halfDir = normalize(lightDir + viewDir);
    float NdotH = max(dot(normal, halfDir), 0.0);
    float NdotL = max(dot(normal, lightDir), 0.0);
    float specPower = (1.0 - roughness) * 156.0 + 16.0;
    float spec = pow(NdotH, specPower);

    // Compose
    float3 color = mix(baseColor, envColor, F * 0.8);
    color = color * (0.5 + 0.7 * pow(NdotL, 1.4)) + baseColor * spec * 1.2;

    // Metallic grain
    float grain = metallicDetail(uv, u.cardType, u.resolution);
    color *= (1.0 + grain * 0.35);

    // Vignette
    float2 vc = float2(0.5 + sin(u.rotate) * 0.15, 0.5 + cos(u.rotate * 0.7) * 0.1);
    float vignette = 1.0 - length((uv - vc) * 1.5);
    color *= mix(0.85, 1.0, vignette);

    return float4(color, 1.0);
}

    Vertex Shader Deep Dive

    1. Block Geometry (6 faces × 6 verts each = 36 verts/block)

    Each block is a box with 6 faces. Per-face vertex positions computed in a switch:

    switch(faceIndex) {
    case 0u: { // TOP — cone-shaped for organic look
        let ux = qv.x - 0.5;
        let uz = qv.y - 0.5;
        let tr = length(vec2f(ux, uz)) * 2.0;
        let cone = 1.0 - tr * 0.35;           // edges dip slightly
        localPos = vec3f(ux * csX, hh * max(cone, 0.0), uz * csZ);
        faceNormal = normalize(vec3f(-ux * 0.8, 1.0, -uz * 0.8)); // tilted normal
    }
    case 1u: { /* BOTTOM */ }
    case 2u: { /* FRONT  */ }
    case 3u: { /* BACK   */ }
    case 4u: { /* RIGHT  */ }
    case 5u: { /* LEFT   */ }
}

    Key insight: The top face isn't flat — it uses a cone function so tall blocks have slightly domed tops, creating a more organic terrain feel.

    2. Isometric ↔ Flat Morphing

    All geometry smoothly transitions between 3D isometric and flat 2D:

    let isoAngleY = mix(ISO_ANGLE_Y, FLAT_ANGLE_Y, progress);  // 0.5 → 0
let isoAngleX = mix(ISO_ANGLE_X, FLAT_ANGLE_X, progress);  // 0.9 → ~π/2
// Cell size morphs from 3D footprint to 2D footprint
var csX = mix(CELL_FOOTPRINT_ISO, CELL_FOOTPRINT_FLAT, progress);

    3. Grid-Based Shadow Casting (Ray March)

    Traces rays from each block toward the light source through the height grid:

    fn gridShadow(myH: f32, col: f32, row: f32) -> f32 {
    let lightDir = normalize(vec3f(0.38, 0.84, 0.36));
    var g = vec2f(lightDir.x, lightDir.z);
    g = g / length(g);

    for (var s: i32 = 1; s < 10; s = s + 1) {   // march 10 steps
        let nc = bc + i32(round(g.x * f32(s)));
        let nr = br + i32(round(g.y * f32(s)));
        let nh = heightAt(nc, nr);
        if (nh > myH + 0.06) {                    // neighbor is taller
            let t = smoothstep(myH + 0.06, myH + 0.36, nh);
            let fall = 1.0 - f32(s) * 0.05;       // shadow fades with distance
            sh *= mix(1.0, 0.78, t * 0.55 * max(fall, 0.5));
        }
    }
    return max(sh, 0.58);  // never fully black
}

    4. Valley Ambient Occlusion

    Blocks surrounded by taller neighbors get darkened:

    fn valleyOcclusion(myH: f32, col: f32, row: f32) -> f32 {
    for dc in -1..1, dr in -1..1:
        maxNeighborHeight = max(maxNeighborHeight, heightAt(bc+dc, br+dr));
    let diff = max(0.0, maxNeighborHeight - myH);
    return smoothstep(0.0, 0.5, diff);
}

    5. Per-Vertex Outputs

    The vertex shader computes and passes to fragment:

    Varying Purpose
    color Block color (green gradient by contribution level)
    shade Diffuse lighting (Lambertian)
    isBase 1.0 if empty cell (level 0)
    isTop 1.0 if top face
    heightFrac 0=bottom, 1=top of block
    topRim Distance from center of top face (for edge darkening)
    shadowCast Pre-computed shadow from taller neighbors
    peakLift Block height (for peak glow)

    valleyAO Ambient occlusion in valleys
    fresnel View-dependent reflectance

    Fragment Shader Deep Dive

    The fragment shader applies 8 post-processing effects:

    // 1. Tone mapping — warm highlights on lit faces
let tone = mix(toneLo, toneHi, pow(shade, 0.78));

// 2. Top face canopy boost
lit = lit * mix(canopy, vec3f(1.0), progress);

// 3. Peak highlight — tall blocks glow
lit = lit * mix(1.0, peakShade, depthFactor * isTop);

// 4. Top rim darkening — edges of top face slightly darker
lit = lit * mix(rimDark, 1.0, progress);

// 5. Contact shadows — base of blocks darkened
let contact = mix(0.88, 1.0, smoothstep(0.0, 0.7, heightFrac));

// 6. Cast shadows from grid ray march
lit = lit * mix(castSoft, 1.0, progress);

// 7. Color temperature — warm on peaks, cool in valleys
let warmShift = vec3f(1.04, 1.01, 0.96);
let coolShift = vec3f(0.97, 0.99, 1.03);

// 8. Film grain — organic texture

vertex VertexOut terrainVertex(
    uint vertexID <span class="xref-missing" title="note not in this site">vertex_id</span>,
    uint instanceID <span class="xref-missing" title="note not in this site">instance_id</span>,
    constant Uniforms &u <span class="xref-missing" title="note not in this site">buffer(0)</span>,
    device const packed_float3* positions <span class="xref-missing" title="note not in this site">buffer(1)</span>,
    device const float* heights <span class="xref-missing" title="note not in this site">buffer(2)</span>,
    device const uint* colors <span class="xref-missing" title="note not in this site">buffer(3)</span>,
    device const float* heightGrid <span class="xref-missing" title="note not in this site">buffer(4)</span>
) {
    VertexOut out;

    uint faceIndex = vertexID / 6;
    uint faceVertex = vertexID % 6;
    float3 blockPos = positions[instanceID];
    float blockHeight = heights[instanceID];

    float blockSize = 0.009;
    float maxHeight = 8.6;
    float h = max(blockHeight * maxHeight, 0.026);
    float height = mix(h, 0.06, u.progress);
    float hh = height * 0.5;

    // Quad vertices (two triangles)
    float2 quadVerts[6] = {
        {0,0}, {1,0}, {0,1}, {0,1}, {1,0}, {1,1}
    };
    float2 qv = quadVerts[faceVertex];

    float csX = mix(0.009, 0.007, u.progress); // cell footprints
    float3 localPos;
    float3 faceNormal;

    // Top face with cone shape
    if (faceIndex == 0) {
        float ux = qv.x - 0.5;
        float uz = qv.y - 0.5;
        float cone = 1.0 - length(float2(ux, uz)) * 2.0 * 0.35;
        localPos = float3(ux * csX, hh * max(cone, 0.0), uz * csX);
        faceNormal = normalize(float3(-ux * 0.8, 1.0, -uz * 0.8));
    }
    // ... other faces (front, back, left, right, bottom)

    // World position
    float3 worldPos = blockPos * blockSize + localPos * blockSize;
    worldPos.y += hh * blockSize;

    // Isometric rotation
    float isoAngleY = mix(0.5, 0.0, u.progress);
    float isoAngleX = mix(0.9, 1.5, u.progress);
    float cy = cos(isoAngleY), sy = sin(isoAngleY);
    float cx = cos(isoAngleX), sx = sin(isoAngleX);

    float ry_x = worldPos.x * cy - worldPos.z * sy;
    float ry_z = worldPos.x * sy + worldPos.z * cy;
    float rx_y = worldPos.y * cx - ry_z * sx;
    float rx_z = worldPos.y * sx + ry_z * cx;

    float scale = u.zoomScale * 3.0; // viewport fitting
    out.position = float4(ry_x * scale / u.aspectRatio, rx_y * scale, rx_z * 0.014 + 0.5, 1.0);

    // Lighting
    float3 light = normalize(float3(0.38, 0.84, 0.36));
    float diffuse = max(dot(faceNormal, light), 0.0);
    out.shade = 0.26 + 0.74 * pow(diffuse, 0.62);
    out.shadowCast = gridShadow(blockHeight, blockPos.x, blockPos.z,
                                heightGrid, GRID_COLS, GRID_ROWS);
    // ... compute remaining varyings

    return out;
}

    Fragment Shader

    fragment float4 terrainFragment(VertexOut in <span class="xref-missing" title="note not in this site">stage_in</span>) {
    float3 bg = float3(0.976, 0.965, 0.945);

    // Tone mapping
    float3 toneLo = float3(0.965, 0.963, 0.958);
    float3 toneHi = float3(1.0, 0.998, 0.993);
    float3 tone = mix(toneLo, toneHi, pow(in.shade, 0.78));
    float3 lit = in.color * tone * (0.94 + 0.06 * in.shade);

    // Contact shadows at block base
    float contact = mix(0.88f, 1.0f, smoothstep(0.0f, 0.7f, in.heightFrac));
    lit *= contact;

    // Cast shadows
    lit *= mix(1.0f, in.shadowCast, 0.58f);

    // Rim light
    lit *= (1.0 + in.rimLight * 0.25);

    // Valley AO
    lit *= (1.0 - in.valleyAO * 0.32);

    // Color temperature by height
    float3 warm = float3(1.04, 1.01, 0.96);
    float3 cool = float3(0.97, 0.99, 1.03);
    lit *= mix(cool, warm, in.heightFrac) * 0.35 + 0.65;

    // Film grain
    float2 p = fract(in.position.xy * float2(0.1031, 0.103));
    p += dot(p, p.yx + 19.19);
    float grain = fract((p.x + p.y) * p.x) - 0.5;
    lit += grain * 0.06;

    return float4(lit, 1.0);
}

    Sphere Waves — Fibonacci Sphere with Perspective Projection

    2000 particles distributed on a Fibonacci sphere with triple-wave displacement, perspective projection, and pinch-to-zoom. CPU-computed via Skia Path on worklet thread.

    How It Works

    The Math: Fibonacci Sphere Distribution

    The Fibonacci sphere places N points nearly uniformly on a sphere surface:

    for (let i = 0; i < N; i++) {
    const a = i / (N * 0.5) - 1.0;              // a ∈ [-1, 1] (latitude)
    const px = cos(i * magicalMul + iTime) * sqrt(1.0 - a * a);
    const py = cos(i * magicalMul + iTime + 11) * sqrt(1.0 - a * a);
    // a = vertical position, px/py = horizontal circle at that latitude
    // sqrt(1 - a²) = radius of circle at latitude a
}

    magicalMul is the golden ratio angle (~2.4 radians). Changing it creates entirely different 3D structures (tap to randomize).

    Triple-Wave Z Displacement

    Three sine waves at different frequencies create organic depth motion:

    const wave1 = sin(i * 0.1  + iTime * 0.7)  * 80;   // slow, large
const wave2 = cos(i * 0.05 + iTime * 0.3)  * 40;   // slower, medium
const wave3 = sin(i * 0.15 + iTime * 1.2)  * 20;   // fast, small
const z = wave1 + wave2 + wave3;

    Why 3 waves: Single wave = boring uniform oscillation. Triple wave creates complex, organic patterns because the frequencies are incommensurate (don't repeat at the same time).

    Perspective Projection

    Real perspective — closer particles are larger:

    const scale = distance / (distance + z);
const x = centerX + px * WIDTH * 0.4 * scale;
const y = centerY + a  * WIDTH * 0.4 * scale;

    distance is controlled by pinch gesture (default 300). Pinching in → extreme fisheye; pinching out → flattens to orthographic.

    Pulsing Radius

    Each particle pulses based on position + time:

    const pulse = sin(i * 0.2 + iTime * 2.0) * 0.3 + 1.0;
const timePulse = sin(iTime * 0.8) * 0.2 + 1.0;
const intensity = (1.0 - abs(py)) / (N * 0.01);
const baseRadius = max(0.5, min(intensity * 20 * scale, 10));
const radius = baseRadius * pulse * timePulse;

    Animated HSL Gradient

    Colors cycle through hue over time:

    const hueShift = (time * 40) % 360;
colors = [
    `hsl(${(340 + hueShift) % 360}, 90%, 70%)`,
    `hsl(${(280 + hueShift) % 360}, 85%, 75%)`,
    `hsl(${(220 + hueShift) % 360}, 95%, 80%)`,
    `hsl(${(160 + hueShift) % 360}, 88%, 72%)`,
    `hsl(${( 60 + hueShift) % 360}, 92%, 78%)`,
];

float3 hsl2rgb(float h, float s, float l) {
    float c = (1.0 - abs(2.0 * l - 1.0)) * s;
    float x = c * (1.0 - abs(fmod(h / 60.0, 2.0) - 1.0));
    float m = l - c * 0.5;
    float3 rgb;
    if      (h < 60)  rgb = float3(c, x, 0);
    else if (h < 120) rgb = float3(x, c, 0);
    else if (h < 180) rgb = float3(0, c, x);
    else if (h < 240) rgb = float3(0, x, c);
    else if (h < 300) rgb = float3(x, 0, c);
    else               rgb = float3(c, 0, x);

}

    particles[id].radius = radius;

}

    Metaball — Blur + Color Matrix Threshold Trick

    Two draggable blobs that merge organically using zero shader code — just Gaussian blur + alpha channel threshold via color matrix. The simplest way to get liquid/metaball effects.

    How It Works

    The Trick

    The entire effect is just two compositing operations:

    1. Draw two overlapping circles
2. Apply heavy Gaussian blur (radius 30)
3. Apply color matrix that cranks alpha contrast

    When two blurred circles overlap, their alpha values add up. The color matrix then thresholds the alpha — anything above a certain value becomes opaque, everything below becomes transparent. This creates the organic "merging blob" shape.

    The Color Matrix

    <ColorMatrix matrix={[
    1, 0, 0, 0, 0,     // R: pass through
    0, 1, 0, 0, 0,     // G: pass through
    0, 0, 1, 0, 0,     // B: pass through
    0, 0, 0, 60, -30,  // A: output = input * 60 - 30
]} />

    The alpha row is the key: alpha_out = alpha_in * 60 - 30

    Input alpha Output alpha Result
    0.0 -30 → 0 Fully transparent
    0.3 -12 → 0 Still transparent
    0.5 0 → 0 Threshold edge
    0.51 0.6 → clamped Starts appearing
    0.6 6 → 1 Fully opaque
    1.0 30 → 1 Fully opaque

    The steep ramp (multiply by 60) creates a sharp cutoff around alpha ~0.5. Where two blurred circles overlap, their alphas sum past this threshold, creating the smooth merge.

        circle.radius1 = Float(radius)
        circle.color0 = CIColor(red: 0, green: 1, blue: 1, alpha: 1)
        circle.color1 = CIColor(red: 0, green: 1, blue: 1, alpha: 0)

        if let output = circle.outputImage {
            combined = output.composited(over: combined)
        }
    }

    // 2. Gaussian blur
    let blur = CIFilter.gaussianBlur()!
    blur.inputImage = combined
    blur.radius = 30

    // 3. Alpha threshold via color matrix
    let matrix = CIFilter.colorMatrix()!
    matrix.inputImage = blur.outputImage
    matrix.rVector = CIVector(x: 1, y: 0, z: 0, w: 0)
    matrix.gVector = CIVector(x: 0, y: 1, z: 0, w: 0)
    matrix.bVector = CIVector(x: 0, y: 0, z: 1, w: 0)
    matrix.aVector = CIVector(x: 0, y: 0, z: 0, w: 60) // alpha *= 60
    matrix.biasVector = CIVector(x: 0, y: 0, z: 0, w: -30) // alpha -= 30

    return matrix.outputImage!
}

fragment float4 metaballFragment(
    float4 position <span class="xref-missing" title="note not in this site">position</span>,
    constant Uniforms &u <span class="xref-missing" title="note not in this site">buffer(0)</span>
) {
    float2 uv = position.xy;

    float d1 = length(uv - u.circle1);
    float d2 = length(uv - u.circle2);

    // Metaball field: sum of inverse-square falloffs
    float field = (u.radius * u.radius) / (d1 * d1 + 1.0)
                + (u.radius * u.radius) / (d2 * d2 + 1.0);

    // Threshold
    float threshold = 1.0;
    float edge = smoothstep(threshold - 0.1, threshold + 0.1, field);

    // Color: sweep gradient based on angle from origin
    float angle = atan2(uv.y, uv.x);
    float t = angle / (2.0 * M_PI_F) + 0.5;
    float3 col = mix(float3(0, 1, 1), float3(0, 0, 1),
                     sin(t * M_PI_F * 2.0) * 0.5 + 0.5);

    return float4(col * edge, edge);
}

    Fractal Glass — Masked Blur Refraction

    A draggable colored circle viewed through a frosted glass panel. The glass region shows a blurred version of the circle while outside the glass the circle is sharp. A 4x4 grid of gradient tiles creates the irregular "fractal" glass pattern.

    How It Works

    Architecture

    Canvas (full screen)
  ├─ Group (clip = glass rect, INVERTED)
  │    └─ Circle (sharp, outside glass)
  │
  └─ Group (clip = glass rect)
       └─ Mask (mode = alpha)
            ├─ mask: FractalGlassMask (4×4 gradient grid)
            └─ child: Circle + Blur(25)

    Two copies of the circle: 1. Outside glass: Clipped with invertClip — renders everywhere EXCEPT the glass rect 2. Inside glass: Clipped TO the glass rect, blurred, alpha-masked by gradient grid

    The Fractal Glass Mask

    A 4×4 grid of rectangles, each filled with a LinearGradient from white → 45% black:

    // Creates 16 gradient tiles (4 cols × 4 rows)

                start={vec(tileX, tileY)}
                end={vec(tileX + tileWidth - 5, tileY + tileHeight - 5)}

    }
}

    This creates varying alpha across the glass — some regions are fully transparent (white = alpha 1 in mask), some are partially (0.45 alpha). This makes the glass look non-uniform, like real textured glass.

struct Uniforms {
    float2 resolution;
    float2 circleCenter;
    float circleRadius;
    float4 glassRect;     // x, y, width, height
    float3 circleColor;
    int tilesPerAxis;     // 4
};

// Fractal glass mask: 4×4 grid of gradients
float glassMask(float2 uv, float4 rect, int tiles) {
    // Check if inside glass rect
    float2 localUV = (uv - rect.xy) / rect.zw;
    if (localUV.x < 0 || localUV.x > 1 || localUV.y < 0 || localUV.y > 1)
        return 0.0;

    // Which tile?
    float2 tileUV = fract(localUV * float(tiles));

    // Diagonal gradient per tile: white (1.0) → 0.55
    float gradient = mix(1.0, 0.55, (tileUV.x + tileUV.y) * 0.5);
    return gradient;
}

// Soft circle SDF
float circleSDF(float2 uv, float2 center, float radius) {
    return length(uv - center) - radius;
}

// Box SDF for glass rect
float boxSDF(float2 uv, float4 rect) {
    float2 center = rect.xy + rect.zw * 0.5;
    float2 halfSize = rect.zw * 0.5;
    float2 d = abs(uv - center) - halfSize;
    return length(max(d, 0.0));
}

fragment float4 fractalGlassFragment(
    float4 position <span class="xref-missing" title="note not in this site">position</span>,
    constant Uniforms &u <span class="xref-missing" title="note not in this site">buffer(0)</span>,
    texture2d<float> blurredTex <span class="xref-missing" title="note not in this site">texture(0)</span>  // pre-blurred circle
) {
    float2 uv = position.xy;
    float2 normUV = uv / u.resolution;

    float circleDist = circleSDF(uv, u.circleCenter, u.circleRadius);
    bool insideGlass = (uv.x >= u.glassRect.x &&
                        uv.x <= u.glassRect.x + u.glassRect.z &&
                        uv.y >= u.glassRect.y &&
                        uv.y <= u.glassRect.y + u.glassRect.w);

    float3 color;

    if (insideGlass) {
        // Sample blurred texture
        constexpr sampler s(filter::linear);
        float3 blurred = blurredTex.sample(s, normUV).rgb;

        // Apply fractal glass mask
        float mask = glassMask(uv, u.glassRect, u.tilesPerAxis);
        color = blurred * mask;
    } else {
        // Sharp circle outside glass
        float edge = smoothstep(1.0, -1.0, circleDist);
        color = u.circleColor * edge;
    }

    return float4(color, 1.0);
}

    Note: The blur requires a two-pass approach — render the circle to a texture, blur it (Gaussian blur shader or MPSImageGaussianBlur), then sample the blurred texture in the main pass.

    Blur Circles — Noise-Driven Organic Motion

    Two circles moving on simplex noise paths — one sharp with a sweep gradient, one blurred — clipped against each other to create a lens/distortion illusion.

    How It Works

    Noise-Driven Position

    Each circle's position is driven by 2D simplex noise, not random:

    // Constants
const FREQUENCY = 1800;   // noise scale (higher = slower movement)
const A = 30;             // amplitude in pixels
const RADIUS = 80;

// Circle 1: normal speed
const x = noise2D(clock / frequency, 0);
const y = noise2D(0, clock / frequency);
cx = amplitude * x + center.x;
cy = amplitude * y + center.y;

// Circle 2: double frequency, triple amplitude
const x2 = secondNoise2D(clock / (frequency * 2), 0);
const y2 = secondNoise2D(0, clock / (frequency * 2));
cx2 = (amplitude * 3) * x2 + center.x;
cy2 = (amplitude * 3) * y2 + center.y;

    Why simplex noise: Random motion is jittery. Sine waves are repetitive. Simplex noise creates smooth, organic, non-repeating paths — like floating in water.

    Two different noise generators (createNoise2D() called twice) produce independent, uncorrelated paths.

    Compositing Stack

    Canvas
  ├─ Circle 1 (sharp, sweep gradient fill, blur mask on edge)
  │    └─ SweepGradient(cyan → magenta → yellow → cyan)
  │    └─ BlurMask(blur: 10, style: normal)
  │
  ├─ Group (clip = Circle 2 path, rounded rect)
  │    └─ Circle 1 AGAIN (clipped to Circle 2's shape)
  │         └─ Same SweepGradient
  │         └─ Blur(blur: 10)
  │
  └─ Circle 2 (blurred outline, acts as clip shape)

    The illusion: Where Circle 2 overlaps Circle 1, you see a blurred version of Circle 1's gradient. Outside the overlap, Circle 1 is sharp. This creates a lens/magnifying glass effect.

    Animation Loop

    clock.value = withRepeat(
    withTiming(20000, { duration: 20000 }),  // 20 seconds to reach 20000
    -1,    // infinite repeat
    true,  // reverse (ping-pong)
);

    The clock ping-pongs between 0 and 20000, which feeds into the noise function. Since noise is continuous, the motion smoothly reverses.

    Spiral — Logarithmic Spiral with Spring Physics

    ~1800 circles arranged on a logarithmic spiral that responds to touch with spring physics. Sweep gradient + blur mask for depth.

    How It Works

    The Math: Logarithmic Spiral

    const logarithmicSpiral = ({ angle, index }) => {
    const a = index / 4;           // distance from center grows with index
    const k = 0.005;               // spiral tightness
    return {
        x: a * Math.exp(k * angle) * Math.cos(angle * index),
        y: a * Math.exp(k * angle) * Math.sin(angle * index),
    };
};

    The classic logarithmic (equiangular) spiral: r = a * e^(kθ)

    • a = index / 4 → each successive circle is farther from center
    • k = 0.005 → very gentle exponential growth
    • angle * index → each circle is at a different angular position, creating the spiral pattern
    • exp(k * angle) → overall scale of the spiral changes with the angle parameter

    Key insight: The angle shared value controls the entire spiral shape. Changing it smoothly morphs the pattern because cos(angle * index) and sin(angle * index) redistribute all points simultaneously.

    Particle Count

    const spiralCircleCount = Math.floor(windowHeight * 1.8);
// On iPhone 15 Pro: ~1800 circles

    Spring Animation

    const TimingConfig = { duration: 3000, dampingRatio: 1 }; // critically damped

// On tap/interaction:
angle.value = withSpring(newAngle, TimingConfig);

    Each of the ~1800 circle coordinates is a makeMutable shared value that updates via useAnimatedReaction when the angle changes. This means all circles animate simultaneously with the spring.

    Rendering

    Canvas
  └─ Path (all circles combined)
       ├─ SweepGradient (color)
       └─ BlurMask (soft edges for depth)

    All circles are added to a single Skia Path and filled with a sweep gradient. The BlurMask softens edges for a depth-of-field effect.

    Distance-Based Sizing

    Circles farther from center are larger (they have higher index). Combined with the blur mask, this creates a sense of depth — center circles are small and sharp, outer circles are larger and softer.

    float4 position <span class="xref-missing" title="note not in this site">position</span>,
    constant Uniforms &u <span class="xref-missing" title="note not in this site">buffer(0)</span>
) {
    float2 uv = position.xy;
    float3 col = float3(0);

    for (uint i = 0; i < u.count; i++) {
        float fi = float(i);
        float a = fi / 4.0;
        float r = a * exp(u.k * u.angle);
        float2 center = u.center + float2(
            r * cos(u.angle * fi),
            r * sin(u.angle * fi)
        );

        float dist = length(uv - center);
        float maxDist = length(u.center);
        float particleR = clamp(length(center - u.center) / maxDist * 6.0, 1.0, 8.0);

        float sdf = dist - particleR;
        if (sdf < 1.0) {
            float alpha = smoothstep(1.0, -0.5, sdf);
            // Sweep gradient color
            float hue = atan2(center.y - u.center.y, center.x - u.center.x)
                       / (2.0 * M_PI_F) + 0.5;
            float3 particleCol = hsl2rgb(hue * 360.0, 0.8, 0.9);
            col = mix(col, particleCol, alpha);
        }
    }

    return float4(col, 1.0);
}

    Performance note: 1800 particles per fragment is very heavy. Use compute shader + instanced rendering for production. The fragment SDF approach is only viable for <200 particles.

    Variations

    The logarithmic spiral formula is easy to modify:

    Change Effect
    Increase k Spiral expands faster
    cos(angle * i * 2) Double spiral (two arms)
    Add sin(time + i * 0.1) to radius Breathing/pulsing
    Replace exp(k*angle) with angle Archimedean spiral (uniform spacing)
    Add z = sin(i * 0.05 + time) + perspective 3D spiral
