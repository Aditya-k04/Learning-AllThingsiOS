    Blender for Shader Prototyping

    Start with zero prior 3D knowledge and finish with an animated procedural water surface with moving caustics — built deliberately so the node graph translates straight into Metal shaders. No modeling, no sculpting, just textures, math, and light. Targets Blender 4.2 LTS+ and the EEVEE Next renderer.

    What you'll be able to do

    2. 02-shading-modes-and-first-material — make it a blue glossy surface.

    How to use this course

    One .blend file carries through all 11 lessons — each page links a starter and an end file (currently placeholders) so you can jump in anywhere. Every lesson takes 20 minutes or less. Keyboard shortcuts appear as chips like Shift+A, but the full menu path is always given too, so you never need to memorize a key. A running 12-glossary collects every term as you meet it.

    Take your time, follow one step at a time, and by the end you'll have real water — and a shader spec to show for it.

    01 — Getting Around Blender (No-Fear Edition)

    By the end you'll have a saved .blend with one flat plane you can orbit around confidently.

    ⏱ ~15 min · beginner · Files: starter → lesson01-start.blend · end → lesson01-end.blend (placeholder)

    What you'll build

    Why this matters

    Before you can build a water shader, you need a surface to put it on and the confidence to move around it. This lesson gives you both: a flat plane (your future water) and the camera moves you'll use in every lesson that follows.

    New concepts

    • Viewport — the big 3D window in the middle where you see and edit your scene.
    • 3D cursor — the little red-and-white target; new objects appear wherever it sits. It starts at the center.
    • Orbit / pan / zoom — the three ways you move your view: spin around (orbit), slide sideways (pan), and move closer/further (zoom).
    • Regions — the panels around the viewport: the Outliner (top-right, a list of everything) and the Properties editor (bottom-right, all the settings).
    • Scene Collection — the top-level folder in the Outliner that holds every object in your scene.

    Step-by-step

    1. Download and install Blender from blender.org → Download. Pick Blender 4.2 LTS (or newer) and run the installer.
    1. Open Blender and close the splash screen by clicking any empty spot. You'll see the default scene: a cube, a light, and a camera. Menu path: (none — it opens automatically).
    1. Orbit the view: hold the middle mouse button and drag. Shortcut: MMB drag.
    1. Pan the view: hold Shift and drag with the middle mouse button. Shortcut: Shift+MMB.
    1. Zoom the view: roll the scroll wheel forward to get closer, backward to pull away. Shortcut: Scroll wheel.
    1. Snap to straight-on views: press Numpad 1 for front, Numpad 3 for right, Numpad 7 for top. Menu path: View → Viewpoint → Front / Right / Top.
    1. Delete the default cube: click it to select it (its outline turns orange), then press X and confirm. Menu path: Object → Delete.
    1. Add your plane: press Shift+A and choose Mesh → Plane. Menu path: Add → Mesh → Plane.
    1. Practice the transform tools on the plane (optional): press S to scale, G to move (grab), R to rotate, then move the mouse and left-click to confirm. Menu path: Object → Transform → Scale / Move / Rotate.
    1. Save your file: press Ctrl+S, pick a folder, name it, and confirm. Menu path: File → Save.

    Checkpoint

    You did it right if you can orbit smoothly around the plane with MMB drag, and pressing Numpad 7 (top view) shows the plane filling most of the viewport as a clean square. Your saved .blend now contains exactly one plane.

    Try it yourself

    Add a second object with Shift+A — try Mesh → Cube — then delete it again with X. Repeating add-then-delete builds muscle memory for the two moves you'll use most.

    Common mistakes

    • Nothing happens when I orbit — you need a real middle mouse button (the scroll wheel clicks down). On a trackpad, enable Edit → Preferences → Input → Emulate 3 Button Mouse and orbit with Alt+Left-drag.
    • My new object appears in a weird spot — objects spawn at the 3D cursor. Reset it to the center with Shift+C before adding.
    • I pressed X and deleted the wrong thing — press Ctrl+Z to undo; Blender undoes many steps back.

    02 — Shading Modes & Your First Material

    By the end you'll have a blue, glossy plane glowing under real lighting in Rendered view.

    ⏱ ~15 min · beginner · Files: starter → lesson02-start.blend · end → lesson02-end.blend (placeholder)

    What you'll build

    Why this matters

    Water is defined by how it reflects and colors light, so your first job is learning to give a surface a color and a shine. This lesson introduces materials and the shading modes you'll flip between constantly while building your water shader.

    New concepts

    • Viewport shading modes — four ways to preview your scene: Wireframe (just edges), Solid (flat grey, fast), Material Preview (shows materials with a studio light), and Rendered (the real, final look).
    • EEVEE Next vs Cycles — the two render engines. EEVEE Next is real-time and fast; Cycles is slower but ultra-realistic. We use EEVEE Next throughout this course.
    • Material — the recipe for how a surface looks: its color, shininess, metalness, and more.
    • Base Color — the main color of the surface.
    • Metallic — how metal-like the surface is (0 = non-metal like water, 1 = raw metal).
    • Roughness — how blurry the reflections are (low = mirror-glossy, high = matte).

    Step-by-step

    1. Open your saved plane file from lesson 1 (File → Open, or your recent-files list). We build directly on that one plane.
    1. Confirm the render engine is EEVEE Next: in Properties → Render Properties (the little TV/camera-back icon), check that Render Engine reads EEVEE Next.
    1. Switch to Material Preview shading: click the third sphere icon in the row at the top-right of the viewport. Menu path: (viewport header → shading sphere icons).
    1. Select the plane (left-click it), then create a material: Properties → Material Properties (the checkered-sphere icon) → New.
    1. Change the Base Color to blue: click the Base Color swatch and pick a blue from the color wheel.
    1. Drag the Roughness slider down toward 0 and watch the plane get glossy and reflective.
    1. Leave Metallic at 0 — water is not a metal, so a value of 0 keeps it reading as a liquid surface rather than polished chrome.
    1. Switch to Rendered shading: click the fourth (rightmost) sphere icon at the top-right of the viewport to see the plane under the scene's real light. Menu path: (viewport header → rightmost shading sphere).
    1. Save your progress with Ctrl+S. Menu path: File → Save.

    Checkpoint

    You did it right if the plane appears clearly blue and glossy in Rendered view, with a visible highlight where the scene light hits it. If it looks flat grey, you're still in Solid shading — switch to the rightmost sphere.

    Try it yourself

    Drag the Roughness slider all the way up to 1.0 and back down to 0.0. Notice how high roughness makes a chalky, matte surface and low roughness makes a sharp mirror — this single slider will do a lot of work in your water shader.

    Common mistakes

    • My plane stays grey no matter what — you're in Solid shading. Click the third or fourth sphere icon (Material Preview or Rendered) to see materials.
    • The New button does nothing — the plane wasn't selected. Left-click the plane first so it has an orange outline, then click New.
    • It looks like shiny metal, not water — your Metallic slider crept above 0. Set Metallic back to 0 for a water-like surface.

    03 — Meet the Shader Editor (Nodes Explained)

    By the end you'll be able to add a node, connect it, and explain what the two default nodes do.

    ⏱ ~15 min · beginner · Files: starter → lesson03-start.blend · end → lesson03-end.blend (placeholder)

    What you'll build

    Why this matters

    Everything advanced in this course — procedural water, ripples, caustics — is built by wiring nodes together in the Shader Editor. This lesson is pure node literacy: no new look, just the skill of adding and connecting nodes so nothing feels mysterious later.

    New concepts

    • Node — a single box that does one job (holds a color, makes a texture, combines things).
    • Socket — a connection dot on a node. Its color = data type: yellow = color, grey = value/float, blue = vector (like a position or direction).
    • Principled BSDF — one big node that describes a whole physical material (the color, metallic, and roughness you set last lesson all live here).
    • Material Output — the final node; whatever connects into its Surface socket is what you actually see.
    • Noodles — the curvy wires that carry data from one socket to another. Everything flows left → right into the Output.

    Step-by-step

    1. Open your blue-plane file from lesson 2 and select the plane. We reuse the exact material you built.
    1. Open the Shader Editor. Easiest way: click the Shading workspace tab along the very top of the window. Alternative: change any panel's editor type using the dropdown in its top-left corner and pick Shader Editor.
    1. Identify the two default nodes: Principled BSDF on the left feeding Material Output on the right. Trace the noodle connecting them.
    1. Add a new node to experiment with: press Shift+A and choose Search (or Color → RGB). Menu path: Add → Color → RGB.
    1. Connect a noodle: drag from the RGB node's output socket to the Principled BSDF's Base Color input socket.
    1. Disconnect a noodle: click-drag the end of the noodle off its socket and release over empty space.
    1. Mute a node to prove it does nothing: select the Principled BSDF and press M. A red line crosses it and the plane goes flat/black. Press M again to un-mute. Menu path: Node → Mute.

    The node setup

    Connections: - Principled BSDF.BSDF → Material Output.Surface

    Checkpoint

    You did it right if you can add an RGB node with Shift+A, drag a noodle into Base Color to change the plane's color, and then explain out loud that the Principled BSDF builds the material and the Material Output displays it. Muting the BSDF with M should make the plane go dark.

    Try it yourself

    Add a second node with Shift+A — try Converter → Value — and just leave it floating unconnected. Notice its single grey socket (a float). Recognizing socket colors now will save you confusion when you start plugging textures in.

    Common mistakes

    • The Shader Editor is empty — no material is selected. Left-click the plane in the viewport so its material loads into the editor.
    • My noodle won't stick — you dropped it on the wrong socket type; drag onto the input dot directly, and remember color-to-color and value-to-value connect most cleanly.
    • The plane went black and I can't undo the look — the Principled BSDF is muted. Select it and press M again to un-mute.

    04 — Procedural Textures 101 (Noise, Voronoi, Wave)

    By the end you'll have a material you can swap between three procedural textures — and describe how each one looks.

    ⏱ ~15 min · beginner · Files: starter → lesson04-start.blend · end → lesson04-end.blend (placeholder)

    What you'll build

    Why this matters

    Real water shaders almost never use photos — they build ripples, foam, and caustics out of pure math so the pattern can move and never repeat. Learning these three texture nodes is learning the exact primitives (Perlin, Worley, sine) you'll later write in Metal.

    New concepts

    • Procedural texture — a pattern generated from math, not an image. It's infinite and resolution-free, so you can zoom in forever and it never gets blurry.
    • Noise Texture — a soft, cloudy Perlin-style pattern. Great for gentle water swell and irregular surface detail.
    • Voronoi Texture — a cellular pattern made of packed cells. This is the seed of caustics — those bright web-like light shapes on a pool floor.
    • Wave Texture — evenly spaced ripple bands, made from a sine wave. Think of clean, repeating wavefronts.
    • Scale and Detail — Scale zooms the pattern in or out (bigger number = smaller features); Detail adds finer sub-wrinkles on top of the big shapes.
    • Fac vs Color outputs — Fac is a single grey value (one number, 0–1). Color is full RGB. Grey is often all you need for a heightmap; Color gives you tint.

    Step-by-step

    1. Open the Shader Editor at the bottom of the screen (it should still show your material from Lesson 3). Menu path: Editor Type dropdown (top-left of a panel) → Shader Editor.
    1. Add a Noise Texture node. Menu path: Add → Texture → Noise Texture. Shortcut: Shift+A.
    1. Drag the Noise Texture's Color output into the Principled BSDF's Base Color input, then let go. (You can use Fac instead for a grey version.)
    1. Change the Noise Texture's Scale (try 2 then 10) and Detail (try 2 then 8) and watch the clouds shrink and gain fine wrinkles.
    1. Delete the Noise Texture and add a Voronoi Texture in its place. Menu path: select the node then Delete (X), then Add → Texture → Voronoi Texture (Shift+A). Wire its Color → Base Color. Notice the packed-cell look — this becomes caustics later.
    1. Delete the Voronoi and add a Wave Texture. Menu path: Delete (X), then Add → Texture → Wave Texture (Shift+A). Wire Color → Base Color and see the evenly spaced ripple bands.

    These three are the same math primitives that live in shader code: Noise ≈ Perlin, Voronoi ≈ Worley, Wave ≈ sine. You're learning the vocabulary of every water shader.

    The node setup

    Connections: - Noise Texture.Color → Principled BSDF.Base Color - Principled BSDF.BSDF → Material Output.Surface

    Checkpoint

    You did it right if you can swap the texture node between Noise, Voronoi, and Wave, and describe each in a sentence: Noise = soft clouds, Voronoi = cells, Wave = bands.

    Try it yourself

    Put the Voronoi Texture back and crank its Scale to about 50. Watch the cells shrink into a fine web pattern — that dense web is exactly the shape caustics take.

    Common mistakes

    • Plugging Color into the wrong socket — dropping it onto Metallic or Roughness gives a weird result. Fix: connect the texture's Color to Base Color for now.
    • Scale set too high, so it looks like flat noise — a Scale of 1000 is just static. Fix: keep Scale between about 2 and 20 while learning.
    • Expecting the pattern to move — procedural textures sit still until you drive them with coordinates. Fix: that's Lesson 5; for now, static is correct.

    05 — Coordinates & Mapping (Where Textures Live)

    By the end you'll have a texture you can slide across the plane by dragging one slider — the exact value you'll animate later.

    ⏱ ~15 min · beginner · Files: starter → lesson05-start.blend · end → lesson05-end.blend (placeholder)

    What you'll build

    Why this matters

    A texture doesn't know where it lives until you tell it — and controlling position is how water flows. The single Mapping slider you set up here is the value you'll keyframe in Lesson 8 to make the whole surface drift.

    New concepts

    • Texture Coordinate node — the source that says where to sample a texture. It offers several outputs: Generated (0–1 based on the object's bounds), Object (based on another object's position), and UV (the surface's own flat coordinates).
    • Mapping node — a transform box with Location, Rotation, and Scale for a texture. It moves, spins, and stretches the pattern before the texture reads it.
    • Why UV matters for a flat plane — UV is the plane's own built-in 0–1 surface grid. For a flat plane it's the cleanest, most predictable coordinate to map onto.

    Step-by-step

    1. In the Shader Editor, make sure you have a Noise Texture wired into Base Color (rebuild it from Lesson 4 if needed). Menu path: Add → Texture → Noise Texture. Shortcut: Shift+A.
    1. Add a Texture Coordinate node. Menu path: Add → Input → Texture Coordinate. Shortcut: Shift+A.
    1. Add a Mapping node. Menu path: Add → Vector → Mapping. Shortcut: Shift+A.
    1. Wire it up: Texture Coordinate.UV → Mapping.Vector, then Mapping.Vector → Noise Texture.Vector.
    1. Drag the Mapping Location X slider and watch the noise pattern SLIDE across the plane. (Remember this slider — in Lesson 8 we animate it so the water moves on its own.)
    1. Change the Mapping Scale — set X to 4 and leave Y at 1 — to stretch the ripples directionally, so features become long and thin along one axis.

    The node setup

    Connections: - Texture Coordinate.UV → Mapping.Vector - Mapping.Vector → Noise Texture.Vector - Noise Texture.Color → Principled BSDF.Base Color - Principled BSDF.BSDF → Material Output.Surface

    Checkpoint

    You did it right if dragging the Mapping Location X slider slides the whole texture across the plane in real time.

    Try it yourself

    Set Mapping Scale X to 6 and Scale Y to 1, then drag Location X again. You've just made directional ripples that flow along one axis — the start of a current.

    Common mistakes

    • Wiring Generated instead of UV — on some setups Generated looks fine, but UV is the reliable choice for a flat plane. Fix: connect the UV output specifically.
    • Dragging Location but nothing moves — usually the Mapping.Vector isn't actually plugged into Noise Texture.Vector. Fix: check that middle connection.
    • Confusing Scale with the texture's own Scale — the Mapping Scale stretches direction; the Noise Texture Scale zooms features. Fix: use Mapping Scale for stretch, Noise Scale for zoom.

    06 — From Flat to Rippled: Bump & Normals

    By the end you'll have a flat plane that shows raised ripples which shimmer as you orbit around it.

    ⏱ ~15 min · beginner · Files: starter → lesson06-start.blend · end → lesson06-end.blend (placeholder)

    What you'll build

    Why this matters

    Water reads as water because light bounces off tiny slopes on its surface. Faking those slopes with a Bump node — instead of adding millions of real polygons — is exactly how a fast water shader is built, in Blender and in Metal alike.

    New concepts

    • Height vs Normal — a height is a grey heightmap (bright = high, dark = low). A normal is the direction a surface faces at each point. Light cares about normals, not heights.
    • Bump node — converts a grey height signal into per-pixel normals, so a flat surface acts bumpy to the light without changing its geometry.
    • Strength / Distance — Strength is how strongly the fake bumps affect shading; Distance is the apparent height difference. Small values look like gentle water.
    • ColorRamp — remaps values to control contrast. Placed before the Bump, it sharpens soft noise into crisper ripple edges.
    • How light reacts to normals — as you orbit, the highlight slides across the tilted normals, creating the shimmer that sells the ripple illusion.

    Step-by-step

    1. Start from your Lesson 5 setup: Texture Coordinate → Mapping → Noise Texture → Principled BSDF. Switch the viewport to Rendered shading so you can see lighting. Menu path: Viewport Shading icons (top-right of the 3D view) → Rendered. Shortcut: Z then choose Rendered.
    1. Add a Bump node. Menu path: Add → Vector → Bump. Shortcut: Shift+A.
    1. Feed the Noise into it: wire Noise Texture.Color (or Fac) → Bump.Height, then Bump.Normal → Principled BSDF.Normal. (Bump goes into Normal, never Base Color.)
    1. Adjust the Bump Strength — try 0.2 for calm water, up to 0.6 for choppier — and watch the ripples deepen.
    1. Add a ColorRamp BEFORE the Bump to sharpen contrast. Menu path: Add → Converter → Color Ramp. Shortcut: Shift+A. Wire Noise Texture.Color → ColorRamp.Fac, then ColorRamp.Color → Bump.Height. Slide the black and white stops closer together to crisp up the ripples.
    1. Orbit the view (middle-mouse drag) and watch the highlight travel across the ripples. Menu path: drag with Middle Mouse Button in the 3D view.

    The node setup

    Connections: - Noise Texture.Color → ColorRamp.Fac - ColorRamp.Color → Bump.Height - Bump.Normal → Principled BSDF.Normal - Principled BSDF.BSDF → Material Output.Surface

    Checkpoint

    You did it right if the flat plane now shows raised ripples that shimmer and shift their highlight as you orbit the view.

    Try it yourself

    Drag the ColorRamp's two color stops very close together. The ripples turn into sharp, high-contrast crests — a rougher, more turbulent water look.

    Common mistakes

    • Bump plugged into Base Color instead of Normal (looks flat/wrong) — fix: connect Bump.Normal → Principled BSDF.Normal, and route color separately if you still want it.
    • Strength cranked to maximum — huge Strength makes jagged, plastic-looking spikes. Fix: keep Bump Strength around 0.2–0.5 for believable water.
    • Still in Solid shading, so nothing looks bumpy — Bump only shows with real lighting. Fix: switch the viewport to Rendered shading (Z).

    07 — Build a Water Material

    By the end you'll have a glassy, slightly rippled, see-through water plane sitting over a visible floor.

    ⏱ 20 min · beginner · Files: starter → lesson07-start.blend · end → lesson07-end.blend (placeholder)

    What you'll build

    Why this matters

    This is the moment all the earlier pieces click together — ripples, color, and light passing through glass. Once you can make convincing still water in Blender, translating the same ideas (transmission, index of refraction, roughness) into a Metal shader becomes a matter of copying numbers you already understand.

    New concepts

    • Transmission — a slider that lets light pass through the surface instead of bouncing off. At 1.0 the material becomes see-through, like glass or water.
    • IOR (Index of Refraction) — how much light bends as it enters the material. Water's real-world value is 1.33, which is why a straw looks broken at the waterline.
    • Roughness (for water) — how blurry reflections and refractions are. Calm water is almost mirror-smooth, so we keep this near 0.
    • Base Color — the material's underlying tint. Water is nearly clear, so we use a very faint blue.
    • Raytracing (EEVEE Next) — the setting that lets EEVEE actually compute see-through refraction. Without it, transmission looks flat.

    Step-by-step

    1. Select your water plane, open the Material Properties tab, and find the Principled BSDF. Set Transmission to 1.0 by dragging the slider (or click the field and type 1.0).
    1. Set the IOR field to 1.33. Menu path: Material Properties → Principled BSDF → IOR.
    1. Set Roughness to about 0.05 so reflections stay crisp and mirror-like.
    1. Click the Base Color swatch and pick a very faint blue (nudge the color wheel slightly toward blue, keep it almost white). Menu path: Principled BSDF → Base Color.
    1. Confirm your Lesson 6 bump ripples are still wired into the Normal input of the Principled BSDF (the Bump node feeding Normal). If it came loose, reconnect it.
    1. Add a floor underneath the water. Menu path: Add → Mesh → Plane. Then press G then Z, move the mouse down a little, and click to drop it below the water.
    1. Enable refraction. Menu path: Render Properties → Raytracing — tick the Raytracing checkbox (EEVEE Next needs this for good transmission).
    1. Switch to Rendered shading to judge the result. Shortcut: Z then pick Rendered from the pie menu.

    The node setup

    Connections: - TexCoord.Generated → Mapping.Vector - Mapping.Vector → Noise Texture.Vector - Noise Texture.Fac → ColorRamp.Fac - ColorRamp.Color → Bump.Height - Bump.Normal → Principled BSDF.Normal - Principled BSDF.BSDF → Material Output.Surface

    Checkpoint

    You did it right if you have a glassy, slightly rippled, see-through water plane and you can clearly see the floor plane through it in Rendered shading.

    Try it yourself

    Nudge IOR up to about 1.5 (closer to glass) and watch the floor appear to shift and bend more strongly through the surface — then return it to 1.33 for accurate water.

    Common mistakes

    • Floor is invisible / water looks solid — Raytracing is off. Turn it on in Render Properties → Raytracing.
    • Water looks milky, not clear — Roughness is too high. Bring it down toward 0.05.
    • Base Color too saturated — a strong blue makes water look like paint. Keep the blue almost white and let transmission do the work.

    08 — Make It Move: Keyframe Animation

    By the end you'll have ripples that visibly drift across the water when you press play.

    ⏱ 15 min · beginner · Files: starter → lesson08-start.blend · end → lesson08-end.blend (placeholder)

    What you'll build

    Why this matters

    Static water is only half the story — real water moves. Keyframing teaches you Blender's core animation loop (a value, saved at a frame, interpolated over time), which is the same mental model behind every animated shader parameter you'll write later.

    New concepts

    • Timeline — the strip along the bottom of the window that shows frames and lets you scrub through time.
    • Keyframe — a value you've saved at a specific frame. Blender fills in the frames between two keyframes automatically.
    • Playhead / scrubbing — the vertical marker showing the current frame; dragging it back and forth is called scrubbing.
    • Insert Keyframe (I) — hover over any value field and press I to save that value at the current frame.
    • Graph Editor — an editor that shows keyframes as a curve, so you can see and shape the easing (speed-up / slow-down) between them.

    Step-by-step

    1. Go to frame 1 by dragging the playhead to the far left of the Timeline (or typing 1 in the current-frame field).
    1. Hover your mouse over the Mapping node's Location X field and press I to insert a keyframe. Menu path: right-click the field → Insert Keyframe.
    1. Go to frame 120 by typing 120 in the current-frame field or dragging the playhead to the right.
    1. Change the Mapping Location X to a new value (for example 2.0) so the ripple pattern has shifted.
    1. Hover the Location X field again and press I to insert the second keyframe. Menu path: right-click → Insert Keyframe.
    1. Press Play. Shortcut: Spacebar. Watch the ripples drift.
    1. Peek at the easing. Open a Graph Editor (change an editor's type via its top-left dropdown → Graph Editor) to see the smooth curve between your two keyframes.

    A note on the limitation: keyframes live inside the Timeline and only cover a fixed range — here, frames 1 to 120. Play past the end and the motion stops or jumps back to the start. In Lesson 9 we'll swap this for motion that never ends.

    The node setup

    Connections: - TexCoord.Generated → Mapping.Vector - Mapping.Vector → Noise Texture.Vector - Noise Texture.Fac → ColorRamp.Fac - ColorRamp.Color → Bump.Height - Bump.Normal → Principled BSDF.Normal - Principled BSDF.BSDF → Material Output.Surface - (animated) Mapping.Location X ◆ keyframed at frames 1 and 120

    Checkpoint

    You did it right if pressing Spacebar makes the ripples visibly drift across the water surface.

    Try it yourself

    Change the frame-120 Location X value to a larger number (like 5.0) and the water will appear to flow faster over the same 120 frames.

    Common mistakes

    • Nothing animates on play — you only set one keyframe. You need at least two, at different frames, with different values.
    • Ripples snap back at the end — that's expected: keyframes stop at frame 120. Extend the range or move to drivers (Lesson 9).
    • Pressed I but nothing saved — your mouse wasn't hovering over the Location X field when you pressed it. Hover directly over the field first.

    09 — Endless Motion with Drivers (#frame)

    By the end you'll have water that flows forever, at a speed you control, with zero keyframes.

    ⏱ 15 min · beginner · Files: starter → lesson09-start.blend · end → lesson09-end.blend (placeholder)

    What you'll build

    Why this matters

    Keyframes are great for fixed animations, but real-time shaders need motion that never loops and never stops. A driver computes a value from an expression every single frame — and the #frame expression maps almost exactly to the time uniform you'll feed a Metal shader later.

    New concepts

    • Driver vs keyframe — a keyframe is a value you saved by hand; a driver computes the value from an expression, fresh, every frame. No manual saving.
    • Add Driver — right-click any value field → Add Driver to attach an expression to it.
    • #frame — a built-in expression that equals Blender's current frame number (1, 2, 3, … and it keeps counting up without looping).
    • Scaling speed — multiply #frame by a small number (like * 0.02) to control how fast the value changes.

    Step-by-step

    1. Remove the Lesson 8 keyframes. Right-click the Mapping Location X field → Clear Keyframes.
    1. Right-click the Mapping Location X field again → Add Driver. Menu path: right-click field → Add Driver.
    1. In the driver popover, find the Expression field and type #frame * 0.02. (Set the driver Type to Scripted Expression if it isn't already.)
    1. Press Play. Shortcut: Spacebar. The water now flows endlessly.

    💡 This #frame is exactly the time value you'll feed a Metal shader later (the time uniform). Multiplying by 0.02 here is the same idea as scaling time before you use it in your shader math.

    The node setup

    Connections: - TexCoord.Generated → Mapping.Vector - Mapping.Vector → Noise Texture.Vector - Noise Texture.Fac → ColorRamp.Fac - ColorRamp.Color → Bump.Height - Bump.Normal → Principled BSDF.Normal - Principled BSDF.BSDF → Material Output.Surface - (driven) Mapping.Location X ⚙ driver expression #frame * 0.02

    Checkpoint

    You did it right if the water flows forever at a controllable speed, with zero keyframes on the Location X field — and it never snaps back to the start.

    Try it yourself

    Change the driver expression to #frame * 0.05 for faster flow, or #frame * 0.005 for a slow, lazy drift. Same water, different speed — no re-keying required.

    Common mistakes

    • Field turned purple but nothing moves — the expression is empty or misspelled. It must read exactly #frame * 0.02.
    • Old keyframes still fighting the driver — you didn't clear them first. Right-click → Clear Keyframes, then add the driver.
    • Motion far too fast or slow — your multiplier is off. Tune the 0.02 up or down until the flow feels right.

    10 — Fake Caustics (The Payoff)

    By the end you'll have an animated caustic web dancing across the pool floor under your water.

    ⏱ 20 min · beginner · Files: starter → lesson10-start.blend · end → lesson10-end.blend (placeholder)

    What you'll build

    Why this matters

    Caustics are the single detail that makes water read as water instead of blue glass. Building them by hand teaches you the exact trick real-time engines use — because, as you'll see, nobody actually simulates them in a game.

    New concepts

    • Caustics — the bright, focused net of light you see on a pool floor, made when the wavy surface bends sunlight into concentrated lines. In real-time (games, EEVEE) they are faked with a texture, never physically simulated, because true light-focusing is far too slow to compute per frame.
    • Voronoi cells — a procedural texture that scatters points and draws the regions around them; its edges make natural-looking web/net patterns.
    • Contrast crush — pushing a soft gradient into thin bright veins using a Power math node or a steep ColorRamp, so only the cell edges glow.
    • Emission / Add Shader — Emission makes a surface give off light (glow); Add Shader lets that glow sit on top of the floor's normal color.
    • Projection — placing the pattern onto the floor using its coordinates so the web appears painted across the surface.

    Step-by-step

    1. In the Outliner, click the Floor plane you made in Lesson 7 to select it. Menu path: Outliner → Floor. Shortcut: click the object name.
    1. Open the Shader Editor and make sure it shows the Floor's material. Menu path: Editor Type → Shader Editor. Shortcut: Shift+F3.
    1. Add a Texture Coordinate node. Menu path: Add → Input → Texture Coordinate. Shortcut: Shift+A.
    1. Add a Mapping node and connect the Texture Coordinate's Generated output into its Vector input. Menu path: Add → Vector → Mapping. Shortcut: Shift+A.
    1. Add a Voronoi Texture and connect Mapping's Vector output into its Vector input. Set Scale to about 50 and, if your Feature dropdown offers it, choose Smooth F1 (otherwise leave it on F1). Menu path: Add → Texture → Voronoi Texture. Shortcut: Shift+A.
    1. Add a ColorRamp and connect Voronoi's Distance output into its Fac input, then drag the black and white handles very close together to crush the soft cells into thin bright veins. Menu path: Add → Converter → Color Ramp. Shortcut: Shift+A.
    1. Add an Emission shader and connect the ColorRamp's Color output into its Color input; set Strength to about 3. Menu path: Add → Shader → Emission. Shortcut: Shift+A.
    1. Connect the Emission node's Emission output into the Material Output's Surface input to see the raw web. Menu path: drag from socket to socket. Shortcut: drag the noodle.
    1. Add a driver to animate the pattern: right-click the Mapping node's Location X field and choose Add Driver, then set its expression to frame*0.02. Menu path: Right-click field → Add Driver. Shortcut: right-click.
    1. Press Play and watch the web drift. Menu path: Timeline → Play. Shortcut: Spacebar.

    The node setup

    Connections: - Texture Coordinate.Generated → Mapping.Vector - Mapping.Vector → Voronoi Texture.Vector - Voronoi Texture.Distance → ColorRamp.Fac - ColorRamp.Color → Emission.Color - Emission.Emission → Material Output.Surface

    Checkpoint

    You did it right if when you press play, an animated caustic web dances on the pool floor under the water — thin bright lines that slowly slide and shimmer rather than a flat blue plane.

    Try it yourself

    Add a second Voronoi Texture at a different Scale (try 25) with its own ColorRamp, then blend the two webs together with a Mix node set to Darken (Minimum). Feed the mixed result into the Emission for a richer, layered net.

    Common mistakes

    • The whole floor glows evenly — your ColorRamp handles are too far apart; drag them close together so only the cell edges stay bright.
    • The web is a giant blurry blob — the Voronoi Scale is too low; raise it toward 50 for tight pool-sized cells.
    • The pattern won't move — the driver expression is empty or misspelled; it must be exactly frame*0.02, and the field should turn purple.

    11 — Look-Dev & Prepping to Translate to Code

    By the end you'll have a clean, labeled .blend plus a values cheat-sheet ready to hand to an AI to draft the Metal shader.

    ⏱ 20 min · beginner · Files: starter → lesson11-start.blend · end → lesson11-end.blend (placeholder)

    What you'll build

    Why this matters

    A shader you can't read is a shader you can't port. Judging the look under real lighting and writing every number down is what turns a pile of nodes into a spec you — or an AI — can rebuild in Metal.

    New concepts

    • HDRI / world lighting — an image (or flat color) used as an all-around environment light, so materials are judged under believable light instead of a black void.
    • Node Frame / labeling — a titled box you draw around related nodes with Ctrl+J to group and name them, keeping a big graph readable.
    • Values table — a plain written list of every final number in the material, so nothing lives only inside a node you might lose.
    • Node → shader-function mapping — a one-to-one note of which Blender node equals which code function, so translation is mechanical rather than guesswork.

    Step-by-step

    1. Open World Properties and set a simple environment light — either load a soft HDRI into the Color slot, or use a flat color with Strength around 1. Menu path: Properties → World Properties → Surface → Color. Shortcut: click the World tab.
    1. Switch the viewport to Rendered shading to judge the real look. Menu path: Viewport header → Rendered sphere icon. Shortcut: Z then choose Rendered.
    1. In the Shader Editor, box-select the nodes that make the water surface. Menu path: Select → Box Select. Shortcut: B, then drag.
    1. Press Ctrl+J to wrap them in a Frame, then double-click the Frame header and name it WATER. Menu path: Node → Join in New Frame. Shortcut: Ctrl+J.
    1. Repeat for the caustics nodes: box-select them, press Ctrl+J, and name the Frame CAUSTICS. Menu path: Node → Join in New Frame. Shortcut: Ctrl+J.
    1. Rename each key node by double-clicking its header so the graph reads like a spec (e.g. Voronoi → caustic-cells, Mapping → time-offset). Menu path: double-click node header. Shortcut: F2 with the node selected.
    1. Write down the final numbers in a values table (see below), reading each one straight off its node. Menu path: click each node → read its fields. Shortcut: hover to reveal exact values.

    Values cheat-sheet (fill in your final numbers):

    Parameter Node Value
    Voronoi Scale (caustics) caustic-cells ~50
    Water IOR Principled BSDF ~1.33
    Water Roughness Principled BSDF ~0.05
    Bump Strength Bump ~0.2
    Caustics driver speed time-offset frame*0.02
    Ripple driver speed (Lesson 9 mapping) frame*0.01

    Node → shader-function map:

    Blender node Metal / shader equivalent
    Noise Texture Perlin noise()
    Voronoi Texture Worley / voronoi()
    Mapping offset (driven) a time uniform
    ColorRamp smoothstep() / remap

    The node setup

    Connections: - WATER Frame → contains the ripple Mapping, Noise, Bump, and Principled BSDF feeding Material Output. - CAUSTICS Frame → contains the time-offset Mapping, Voronoi, ColorRamp, and Emission feeding the floor's Material Output. - Mapping:time-offset.Location X ← driver frame*0.02

    Checkpoint

    You did it right if you have a clean, labeled .blend — two named Frames, descriptively renamed nodes, soft world lighting — plus a filled-in values cheat-sheet listing every final number.

    How to hand this to an AI to draft the Metal shader

    • Export the values table above with your real final numbers filled in.
    • Describe each node's role in one line (what it does, not just its name).
    • Name the driver explicitly as the time input (frame*0.02 → time * 0.02).
    • Point the AI at the node → function map so it knows which primitive to reach for.

    Your code destinations, once you're ready to write the shader: - Voronoi in Metal — the caustic cells. - fBm in Metal — the water surface noise. - time-driven motion — replacing #frame with a time uniform.

    Try it yourself

    Duplicate your values table into a plain text note and rewrite each row as a line of pseudo-code (e.g. float cells = voronoi(uv * 50.0);) — a first rough draft of the shader before you ever open an editor.

    Common mistakes

    • Judging the look in Solid mode — colors and glow lie without lighting; always switch to Rendered under a world light before deciding.
    • Framing without naming — an unlabeled Frame helps nobody; always rename the Frame and its key nodes.
    • Trusting your memory for values — numbers vanish the moment you tweak a node; write every final value down in the table.

    Glossary

    Every "New concept" term from the course, grouped by the lesson where it first appears, each defined in one plain-English sentence. Skim it any time a word stops making sense.

    Lesson 1 — Getting around Blender

    viewport (L1)

    The main 3D window where you see and move around your scene.

    3D cursor (L1)

    A crosshair marker showing where newly added objects will appear.

    orbit / pan / zoom (L1)

    The three ways to move your view — rotate around, slide sideways, and move closer or further.

    UI regions (L1)

    The panels that make up the window — the viewport, the Outliner (object list), and the Properties editor (settings).

    scene collection (L1)

    The top-level container in the Outliner that holds everything in your scene.

    Lesson 2 — Shading modes & first material

    viewport shading modes (L2)

    The four preview styles — Wireframe, Solid, Material Preview, and Rendered — that trade speed for realism.

    EEVEE Next vs Cycles (L2)

    EEVEE Next is Blender's fast real-time renderer (what this course uses); Cycles is the slower, physically accurate one.

    material (L2)

    The recipe describing how a surface looks — its color, shine, and more.

    Base Color (L2)

    The main flat color of a material before lighting effects.

    Metallic (L2)

    A slider setting how metal-like a surface behaves, from 0 (non-metal) to 1 (full metal).

    Roughness (L2)

    A slider setting how blurry or sharp reflections are — low is glossy, high is matte.

    Lesson 3 — Shader Editor nodes

    node (L3)

    A single box in the shader graph that does one job.

    socket (L3)

    A connection dot on a node; its color shows its type — yellow = color, grey = value, blue = vector.

    Principled BSDF (L3)

    The all-in-one main shader node that handles most real-world surfaces.

    Material Output (L3)

    The final node that sends a material to the renderer.

    noodle (L3)

    The curved wire connecting one socket to another.

    mute (L3)

    Temporarily switching a node off without deleting it.

    Lesson 4 — Procedural textures 101

    procedural texture (L4)

    A pattern generated by math instead of a painted image, so it never pixelates.

    Noise Texture (L4)

    A procedural texture making soft, cloud-like random variation.

    Voronoi Texture (L4)

    A procedural texture that scatters points and draws cell-like regions around them.

    Wave Texture (L4)

    A procedural texture making bands, rings, or ripples.

    Scale (L4)

    A setting controlling how large or small a texture's pattern appears.

    Detail (L4)

    A setting adding finer layers of variation on top of a texture.

    Fac vs Color (L4)

    Two outputs of a texture — Fac is a single grey value, Color is full RGB.

    Lesson 5 — Coordinates & mapping

    Texture Coordinate (L5)

    A node providing the position system a texture is placed on — Generated, Object, or UV.

    Mapping node (L5)

    A node that moves, rotates, and scales a texture via its Location, Rotation, and Scale inputs.

    UV (L5)

    A flat coordinate layout describing how a texture wraps onto a surface.

    Lesson 6 — Bump & normals

    height vs normal (L6)

    Height is how high a fake bump sticks up; a normal is the surface direction the light bounces off.

    Bump node (L6)

    A node that turns a grey height pattern into fake surface bumpiness.

    Strength / Distance (L6)

    Bump settings controlling how strong and how deep the fake bumps appear.

    ColorRamp (L6)

    A node that remaps input values to a custom gradient, often to sharpen or tint them.

    normals (L6)

    The invisible surface directions that decide how light reflects across a shape.

    Lesson 7 — Build a water material

    Transmission (L7)

    A setting letting light pass through a surface, making it see-through like glass or water.

    IOR (L7)

    Index of Refraction — how much light bends passing into a material (water is about 1.33).

    Roughness (for water) (L7)

    Kept very low here so the water stays glassy and reflective rather than foggy.

    Lesson 8 — Keyframe animation

    Timeline (L8)

    The strip along the bottom showing frames and playback controls.

    keyframe (L8)

    A saved value at a specific frame; Blender fills in the frames between two keyframes.

    playhead / scrubbing (L8)

    The marker showing the current frame; dragging it back and forth is scrubbing.

    insert keyframe (I) (L8)

    The action of recording a value at the current frame, done with the I key.

    Graph Editor / easing (L8)

    An editor showing animation as curves, where you shape the acceleration (easing) between keyframes.

    Lesson 9 — Drivers & #frame

    driver (L9)

    A tiny formula that sets a value automatically instead of you keyframing it.

    Add Driver (L9)

    The right-click command that attaches a driver to a field.

    #frame (L9)

    A built-in variable equal to the current frame number — the course's stand-in for a time uniform.

    speed scaling (L9)

    Multiplying #frame by a small number to control how fast the driven motion runs.

    Lesson 10 — Fake caustics

    caustics (L10)

    The bright light-web on a pool floor, faked with a texture in real-time renderers instead of physically simulated.

    Voronoi cells (L10)

    The cell regions of a Voronoi texture whose edges form the caustic web.

    Power / contrast crush (L10)

    Pushing a soft gradient into thin bright veins with a Power node or a steep ColorRamp.

    Emission / Add Shader (L10)

    Emission makes a surface glow; Add Shader lays that glow on top of the base material.

    projection (L10)

    Placing the caustic pattern onto the floor using its coordinates so it reads as painted across the surface.

    Lesson 11 — Look-dev & translate to code

    HDRI / world lighting (L11)

    An image-based environment light used to judge materials under believable all-around light.

    node Frame / labeling (L11)

    A titled box drawn around related nodes with Ctrl+J to keep a big graph readable.

    values table (L11)

    A written list of every final number in a material, so nothing lives only inside a node.

    node → shader-function mapping (L11)

    A one-to-one note of which Blender node equals which code function, making translation mechanical.
